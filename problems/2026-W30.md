## 課題カード (3枚)

### 課題: 規制当局が「ML 方策の合否試験そのもの」を公募し始めたが、"objective・practicable・repeatable" な試験法が誰も持っていない

- **誰が困っているか / 深刻さ**: NHTSA 自身、そして全 AV 事業者。NHTSA は「メーカーが自車の適合を**確実に**判定できる」客観基準を作ると明言した ([Bloomberg 7/16](https://www.bloomberg.com/news/articles/2026-07-16/us-takes-aim-at-autonomous-car-mishaps-in-safety-rulemaking-push))。これは W28-29 で追ってきた「誰が性能を検証するか」が、規制の力で**試験法の標準化**という具体形に固まった瞬間。数百億ドル規模の展開許認可が、この試験仕様の中身に直結する。
- **なぜ未解決か**: ML で学習した挙動は本質的に確率的で、"repeatable"(同じ入力で同じ結果) と "objective"(主観に依らぬ合否) を素朴には満たさない。従来の機能安全 (ISO 26262 — 部品故障を扱う規格) は「壊れないこと」を測るが、「正しく判断するか」を測る道具立てがない。NHTSA も「各 competency に metrics と評価手段を紐付ける」段階=**まだ手段が存在しない**ことを認めている ([TT News](https://www.ttnews.com/articles/nhtsa-autonomous-rulemaking))。技術とルールの両方が空白。
- **既存プレイヤーと限界**: Applied Intuition / Foretellix 等は「シナリオを回す」基盤は持つが、「客観・再現・実施可能」を満たす合否 metric の設計は未確立 (推測)。UL 4600 (safety case の書式規格) は"書き方"を定めるだけで合否装置がない (W28 カード3 と同じ限界)。
- **対象者の技術との接続**: **ど真ん中が P1 (planner 評価・HIL)**。「確率的方策に対して repeatable かつ objective な合否指標をどう定義するか」は、そのまま論文の問い (evaluation methodology) になり、規制の需要と学術貢献が一致する稀なケース。behavioral competency の分解は VLA (Vision-Language-Action、視覚言語入力から行動を出す基盤モデル) の能力評価にも横展開可能。
- **最初の一歩 (1週末)**: NHTSA が過去に列挙した AV behavioral competency リスト (2017 の "A Framework for Automated Driving System Testable Cases" 等の公開資料) を1つ取り、各項目に「客観・再現・実施可能を満たす試験が現状あるか」を ○/△/× で採点した1枚の表を作る。× が集中する箇所が研究の空白。

### 課題: 倉庫自動化の bottleneck は「全自動」ではなく「自分が分からない時を分かる」能力 — 実顧客を持つ企業が資金繰りで死ぬ

- **誰が困っているか / 深刻さ**: 導入側の倉庫事業者と、Attabotics のように**実顧客 (Nordstrom) を持ちながら死んだ**サプライヤー。倉庫の76%が人手不足を報告 ([SCMR](https://www.scmr.com/article/why-fully-autonomous-warehouses-are-still-out-of-reach-for-most))。にもかかわらず現実の運用は "hybrid autonomy" — ロボットが**低信頼度の判断を人間にescalate**する方式が主流で、full autonomy は依然「大半にとって手の届かない」状態 ([SCMR](https://www.scmr.com/article/why-fully-autonomous-warehouses-are-still-out-of-reach-for-most))。Attabotics 破産・iRobot 危機は、記録的な humanoid 資金 (今週 Humanoid $152M) と好対照で、**資本の向かう先と現場が必要とする能力がズレている**兆候 (推測)。
- **なぜ未解決か**: hybrid の価値は「いつ人間を呼ぶか」の判断精度に依存する。過剰にescalate すれば人件費が消えず ROI が出ず、過少なら誤作動する。これは calibration (信頼度が実際の正解率と一致しているか) と OOD 検知 (学習分布外の状況の検出) の問題そのもの。だが多くの現場ロボットは policy の**信頼度較正を測る仕組みを持たない** (推測)。加えて Attabotics 型の重厚な自社ハード (立体構造) は資本集約的で、閾値を越える前に資金が尽きる。
- **既存プレイヤーと限界**: Locus/6 River 等の AMR (自律搬送ロボット) は搬送に特化し escalation は運用でカバー。humanoid 各社はデモ成功率を語るが「不確かな時に手を止められるか」の指標は公開しない (推測、W29 カード2 の主張-証拠ギャップと同根)。
- **対象者の技術との接続**: 「policy が"分からない"を検知して安全に人間へ渡す」評価は P1 の評価技術 + uncertainty/OOD の交差点。エッジで安く回すための **FM 蒸留 (P2)** は escalation コスト側の実装条件。VLA の calibration は研究フロンティア。
- **最初の一歩 (1週末)**: 公開 manipulation/AMR データセットで policy 1つを走らせ、信頼度スコアと実際の成功/失敗の相関 (reliability diagram) を1枚描く。「較正が崩れる状況=escalate すべき状況」を可視化できれば hybrid の核を掴んだことになる。

### 課題: 自動車 Tier-1 が humanoid に賭け始めたが、「人の隣で学習した手が動く」ことの安全検証枠組みがない

- **誰が困っているか / 深刻さ**: 工場に humanoid を入れたい製造業と、それを担ぐ Tier-1 サプライヤー。今週の Humanoid $152M には **Bosch・Schaeffler** という自動車部品の巨人が出資し、産業用 humanoid の量産 (beta Q4 2026) を狙う ([SiliconANGLE 7/21](https://siliconangle.com/2026/07/21/humanoid-raises-152m-1-35b-valuation-bring-human-like-robots-factories/))。自動車で培った量産・検証の知見が robotics に流れる構造転換だが、対象が「車の運転」から「人の隣での接触を伴う manipulation」に変わる。
- **なぜ未解決か**: 自動車の機能安全 (ISO 26262) や AV の behavioral competency (カード1) は**運転**を前提とする。工場 humanoid は接触リッチ (contact-rich、物や人に触れて力を加える) な作業を、人間の同僚のすぐ隣で行う。「学習された手が想定外の力を出さないか」を測る標準試験は存在せず、産業用ロボットの既存規格 (ISO 10218、柵で人を隔離する前提) は協働・学習方策を想定していない (推測)。
- **既存プレイヤーと限界**: 協働ロボット (cobot) の安全規格 (ISO/TS 15066) は速度・力の上限を定めるが、VLA のような学習方策が"何をしようとしているか"の評価は範囲外 (推測)。humanoid 各社はデモ動画が中心で第三者の安全検証プロトコルは見当たらない。
- **対象者の技術との接続**: **P3 (VLA/E2E) に3枚中で最も近い**。「学習した manipulation 方策の安全性を、行動を実行する前に評価する」問題は VLA 評価の最前線で、P1 の評価技術を運転から manipulation へ横展開する筋。カード1 の "behavioral competency" 概念を manipulation ドメインに翻訳する研究は新規性が高い。
- **最初の一歩 (1週末)**: 公開 VLA (例: OpenVLA 系) を1つ選び、その出力する行動系列に対して「接触前に危険 (過大な力・人との衝突) を予測できる安全 monitor」を後付けする最小実験の設計メモを1枚書く。まず"評価だけ"を切り出す。

## ストックへの示唆

- **反復テーマ (3週連続) が「規制による試験標準化」へ収束した。** W28「誰が検証するか」→ W29「NHTSA が月末期限で証拠を要求」→ **W30「NHTSA が試験法そのものを rulemaking で公募し、objective・practicable・repeatable な合否試験を作ると宣言」**。抽象的な問いが、3週間で「連邦の試験仕様策定」という具体的なイベントに落ちた。これは追跡してきた仮説の最も強い裏付け。
- **新しい軸: 評価ドメインが「運転」から外へ染み出した。** 今週の3枚は driving-behavior 試験法 / warehouse の escalation 較正 / manipulation 安全検証 と、物理モダリティは違うが**すべて「学習方策の合否をどう客観的に測るか」の別バリエーション**。W29 で芽が出た fleet-level に続き、評価問題が複数ドメインに分岐し始めた。
- **推し (3週連続で不動): 「自律システムの独立評価 / 試験標準 (evaluation・safety case・testable competency)」を深掘りドメインに確定してよい段階。** 理由: (1) P1 業務と完全一致し Phase 1 の論文の種と二重使用可、(2) 規制が需要を**試験仕様のレベルまで**具体化した、(3) 資金は「作る側」(humanoid) に殺到し「測る側」は空いている (W28-30 で一貫)。
- **次の一手 (期限イベント)**: NHTSA の月末 (7月末) 期限で各社が提出する first responder 対応策と、rulemaking の public comment 開始が W31-32 に見えるはず。**各社が evaluation の何を持ち何を持たないかが公開情報として一斉に出る、年に一度級の観測機会**。W31 で必ず追跡すること。
