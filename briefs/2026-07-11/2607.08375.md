# WCog-VLA: 意味論的予測と生成的 world model を両立する E2E 自動運転 VLA

- **arxiv-id**: 2607.08375 ／ **topic**: next_arch ／ **project**: P3 ／ **relevance**: 5/5 ／ **hf_upvotes**: 0
- **URL**: http://arxiv.org/abs/2607.08375v1

## 何を解決する論文か
既存の E2E 自動運転向け VLA (Vision-Language-Action; 視覚+言語入力から行動を出すモデル) は、世界の包括的な認知を持たないか、世界予測が断片的で、結果として reactive (目の前の状況に反応するだけ) な運転に留まる。WCog-VLA は「意味レベルの世界予測」と「生成レベルの世界進化」を 1 つの VLA に統合し、proactive (先読みして動く) な運転を目指す。planning ベンチマーク NAVSIM で SOTA の PDMS 92.9 を報告。

## 手法の核心
- **dual-level の world cognition**: 意味レベルと生成レベルの 2 階建て。意味レベルでは 3D 空間認識を取り込み、agent token (周辺車両・歩行者などの動的主体を表す専用トークン) で world dynamics を捕捉する。
- **Game-CoT (Game-theoretic Chain-of-Thought)**: CoT (Chain-of-Thought; 推論過程を明示的に言語化させる技法) をゲーム理論的な相互作用推論 (「相手がこう動くなら自分はこう動く」) に特化させた形式。85k 件の Game-CoT アノテーション付き大規模データセットも構築。
- **ADDT (Aligned Decoupled Diffusion Transformer)**: 生成レベルの world model。物理的に妥当な複数エージェントの joint trajectory を diffusion model (ノイズ除去を繰り返して生成する生成モデル) で合成する。scene representation alignment により denoising ステップ数を減らし、推論を大幅に高速化。
- **NAVSIM で SOTA**: PDMS (Predictive Driver Model Score; 衝突・走行可能領域逸脱・快適性などを合成した closed-loop 風の planning 評価指標) 92.9。

## P3 (次世代アーキテクチャ) への適用可能性
relevance_criteria の高優先 3 項目 (VLA の構造・学習、world model、最新 E2E AD アーキテクチャ) を全て踏む今日の本命。「VLA に world model をどう埋め込むか」に対して、意味 (トークン・言語推論) と生成 (diffusion による軌道進化) を分けて持たせ alignment で繋ぐ、という設計は P3 のアーキテクチャ検討の直接の参照点になる。周辺エージェントとの相互作用を CoT データとして明示的に教える路線も、interaction-aware planning の学習法として要注目。

### 試すべき実験アイデア
1. **agent token の効果検証**: 自社 E2E/VLA プロトタイプに周辺エージェント専用トークンを追加し、interaction が濃いシナリオ (合流・交差点) での PDMS/衝突率の変化を測る。scene 全体を一様にトークン化する現行設計との比較。
2. **Game-CoT 風データの小規模生成**: 手持ちの走行ログから「相手の意図仮説 → 自車の応答」形式の CoT アノテーションを LLM で自動生成し、fine-tuning に混ぜて reactive な失敗 (譲り合いデッドロック等) が減るか見る。
3. **denoising 削減の alignment 再現**: 軌道生成 diffusion を使っているなら、scene representation alignment に相当する条件付け強化でステップ数をどこまで削れるか (品質 vs レイテンシ) を測る。実車推論予算に直結する。
