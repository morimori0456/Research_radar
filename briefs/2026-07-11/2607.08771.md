# ZipDepth: 6.1M パラメータで foundation model 級 zero-shot 単眼深度をエッジに載せる

- **arxiv-id**: 2607.08771 ／ **topic**: fm_distill_finetune ／ **project**: P2 ／ **relevance**: 5/5 ／ **hf_upvotes**: 0
- **URL**: http://arxiv.org/abs/2607.08771v1

## 何を解決する論文か
monocular depth estimation (単眼カメラ画像 1 枚から距離を推定するタスク) は Depth Anything 系の foundation model で zero-shot 汎化 (未知ドメインでも再学習なしに動く性質) が実現したが、計算量が大きく組込み・モバイルでは動かない。一方で軽量モデルは単一ドメインの self-supervised 学習が中心で、ドメインが変わると静かに壊れる。ZipDepth は foundation model からの大規模 knowledge distillation (大モデル teacher の出力を教師信号として小モデル student を学習する圧縮手法) で、6.1M パラメータのまま zero-shot 汎化を獲得する。

## 手法の核心
- **reparameterizable な encoder-decoder**: 学習時は複数分岐の構造で表現力を確保し、推論時には分岐を数学的に単一の畳み込みへ畳み直す reparameterization (RepVGG 系の技法) を採用。推論グラフを単純化し、エッジデバイスで実時間動作する。
- **foundation model teacher × multi-domain 大規模データ**: 単一ドメインでの学習ではなく、大規模な multi-domain 訓練セット上で foundation model の予測を蒸留する。「何を教えるか」だけでなく「どんな分布の入力で教えるか」が zero-shot 性の源泉、という設計。
- **50 倍のパラメータ差を埋める capacity gap 対策の実例**: 5 つのベンチマークで、zero-shot 精度と deployment 効率のトレードオフにおいて軽量モデル中最良。teacher (50x 大) の精度に大きく迫る。
- **サーバ GPU から電力制約デバイスまで実時間**: 単一アーキテクチャで幅広いハードウェアをカバー。

## P2 (FM 蒸留 + 適合) への適用可能性
relevance_criteria の本命 (再現可能な蒸留 recipe、capacity gap 対策)。「foundation model の頑健性を、multi-domain データ経由の蒸留で小モデルに移す」という recipe は深度に限らず、検知・セグメンテーションなど他の dense prediction タスクの蒸留にそのまま横展開できる。Jetson 級のエッジ推論を前提とする案件では、reparameterization + 蒸留の組み合わせは設計テンプレートとして直接使える。

### 試すべき実験アイデア
1. **蒸留データ分布の ablation 再現**: 自社の蒸留パイプラインで「teacher 固定・蒸留データを単一ドメイン → multi-domain に広げる」比較を行い、student の out-of-domain 頑健性がデータ分布でどれだけ動くかを定量化する。ZipDepth の主張の最重要部分の検証。
2. **Jetson 上での実測**: 公開されれば ZipDepth を Jetson (Orin 等) で TensorRT 化し、レイテンシ/精度を自社軽量深度モデルと比較。reparameterization が TensorRT 最適化とどう噛み合うかも確認する。
3. **他タスクへの recipe 転用**: 同じ「foundation teacher + multi-domain 蒸留 + reparameterizable student」構成を BEV 特徴抽出や物体検知の backbone 蒸留に適用してみる。
