# Wat3R: アノテーションゼロで 3D 再構成モデルを別ドメイン (水中) へ適合させる

- **arxiv-id**: 2607.08772 ／ **topic**: fm_distill_finetune ／ **project**: P2 ／ **relevance**: 4/5 ／ **hf_upvotes**: 0
- **URL**: http://arxiv.org/abs/2607.08772v1

## 何を解決する論文か
feed-forward 3D reconstruction モデル (DUSt3R 系のように、複数画像から 1 回の順伝播で深度・点群を出すモデル) は地上 (in-air) の大量アノテーションで学習されており、光の減衰・散乱が激しくラベルがほぼ存在しない水中では性能が落ちる。Wat3R は、ラベルなしの実水中動画だけを使う cross-domain semi-supervised learning (ラベル付き source ドメイン + ラベルなし target ドメインで学習する枠組み) で、in-air モデルを水中へ適合させる。水中は極端な例で、本質は「アノテーション不能なドメインへの foundation model 適合 recipe」。

## 手法の核心
- **teacher-student によるラベルフリー適合**: in-air 学習済みモデルを teacher とし、ラベルなし水中動画への予測を教師信号として student を学習する。target ドメインのアノテーションを一切使わない。
- **cross-view consistency loss**: 水の減衰・散乱で現在ビューの情報が劣化していても、他ビューの幾何的手がかり (同じシーンを別角度から見た整合性) で補償する loss を設計。「劣化ドメインでは単一入力の教師信号が信用できない」問題への一般的な処方。
- **評価ベンチマーク Water3D の同時構築**: 適合先ドメインに評価基盤がない場合はまず作る、という実務的にも正しい手順。多様な水域・水中シナリオをカバーし、multi-view depth と点群再構成で SOTA を確認。
- **コード・データ公開**: https://github.com/LSXI7/Wat3R

## P2 (FM 蒸留 + 適合) への適用可能性
relevance_criteria の「少データでの適合・ドメインギャップの克服・基盤モデルを別ドメインに転用する recipe」に直球。自動運転文脈では「水中」を「悪天候 (雨・霧・夜間)」「別センサ (異なるカメラ・レンズ特性)」「別地域」に読み替えられる。ラベル付与が高コストなドメインへ深度・幾何系 foundation model を持っていく際の、teacher-student + 幾何整合性 loss という構成はそのまま流用できる。

### 試すべき実験アイデア
1. **悪天候適合への転用**: in-air ↔ 水中の関係を「晴天 ↔ 雨天/夜間」に置き換え、晴天学習済みの深度/3D モデルを、ラベルなしの悪天候走行動画で teacher-student 適合させる。cross-view consistency は連続フレーム + エゴモーションで代替可能か検証する。
2. **teacher の信頼度重み付け**: 劣化が強い領域 (水中なら遠方、雨天なら反射面) で teacher 予測の重みを下げる uncertainty 重み付けを足し、素の pseudo-label 蒸留との差を測る。
3. **ミニ Water3D 方式の評価セット構築**: 適合先ドメイン (例: 特定カメラの夜間データ) に対し、少数の高信頼参照 (LiDAR 等) で小さな評価セットを先に作る運用を標準化する。
