# Streaming Multi-Agent Autoregressive Diffusion Model with World State Registers（WorldWeaver）

**topic:** next_arch (P3) ・ **arXiv:** 2607.21594v1 ・ **published:** 2026-07-23 ・ hf_upvotes: 9

## 何を解決する論文か
multi-agent の interactive world model(複数エージェントが同じ世界で動く、行動に応じた映像生成モデル)は、見た目が整合するだけでなく **「世界の状態」がエージェント間・視点間で共有され、時間発展する**必要がある。既存の autoregressive video diffusion(過去フレームを条件に次を拡散生成する方式)は観測履歴を条件付けに持ち越すだけなので、multi-agent/multi-view で **共有状態を保つのが難しい**。WorldWeaver (W²) は、履歴の持ち越しではなく **明示的な「世界状態レジスタ」**で共有状態を管理する。

## 手法の核心
- **world state registers:** 世界の共有情報を蓄える **learnable token(学習可能な記憶スロット)**。各エージェントの状態も追跡し、**チャンク生成のたびに動的に更新**される。履歴を長々と条件付けする代わりに、圧縮された永続状態として持つ。
- **多面的な supervision で register を接地:** 個々のエージェント状態、**BEV (Bird's-Eye View; 真上からの俯瞰図)** を含む global state view、scene text(場面の説明文)——複数の教師信号で register を「意味を持つ状態」に固定。
- **Mixture-of-Transformers 構造:** **world state のモデリング**と**視覚フレームのモデリング**を別々の重みに分離した MoT 設計(専門化した2系統の Transformer)。状態推論と描画を分業させる。
- **検証:** 2エージェントの Minecraft 動画生成で、明示的 world-state モデリングが **論理的整合性と生成品質の双方を改善**。

## 追っているプロジェクトへの適用可能性
- **P3(本命):** driving world model で「長い観測履歴を条件付けに引きずる」現行方式の代わりに、**永続的な world state register に圧縮して持つ**という設計選択肢。マルチエージェント(周辺車両)・マルチ視点(自車/俯瞰)で状態を一貫させたい我々の設定に直結。BEV supervision で register を接地する点は AD と相性が良い。
- **P3(構造):** MoT の「状態推論と描画で重みを分ける」分業は、long-horizon rollout での drift 抑制・計算配分の観点でも着想になる。
- **P1(横断):** register が保持する global/BEV 状態を planner 評価の「世界の真値プロキシ」として使えれば、closed-loop 評価の信頼性向上に効く可能性。

### 試すべき実験
1. 我々の driving 生成モデルに **world state register を数個〜数十個**足し、BEV と自車状態で supervise。履歴条件付けのみの baseline と、**周辺車両の状態一貫性(消失/瞬間移動の頻度)**で比較。
2. **MoT(状態/描画で重み分離) vs 単一 Transformer** を同パラメータ数で比較し、long-horizon(30–60秒)rollout の drift と計算コストを測る。
3. register 更新頻度(チャンク長)を振り、**状態の鮮度 vs 生成品質**のトレードオフを把握。マルチ視点間で register を共有した時の視点整合性も確認。
