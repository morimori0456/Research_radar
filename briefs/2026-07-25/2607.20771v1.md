# Emergent Compositional Skills in Mixture-of-Experts VLAs

**topic:** next_arch (P3) ・ **arXiv:** 2607.20771v1 ・ **published:** 2026-07-22 ・ hf_upvotes: 0

## 何を解決する論文か
ロボットの汎用方策を組む時、「タスクをどう部分技能に分解するか(階層・サブゴール)」を人手で設計するのが普通。本論文は、**分解の定義を一切与えず**、専門家のデモから end-to-end に学んだ VLA (Vision-Language-Action; 画像・言語から行動を出す方策)が、**自発的に(emergent)再利用可能な原始技能へ分解**できるかを問う。答えは Yes——**Mixture-of-Experts (MoE; 入力ごとに一部の「専門家」サブネットだけを選んで使う疎な構造)** の action head を使うと、router が高レベルの手順制御を、expert が低レベルの原始動作を担うよう自然に役割分担する。

## 手法の核心
- **簡略化した MoE action head:** VLA の行動生成ヘッドを MoE 化。事前のタスク分解・階層は与えず、デモから素直に学習させるだけ。
- **expert の再利用が創発:** 学習された expert は **複数タスクをまたいで多用**され、それぞれが **質的に異なる低レベル動作**に一貫対応。router が実質「高レベルのシーケンシング(どの原始技能をいつ使うか)」を暗黙に学ぶ。
- **性能は据え置き・解釈性は向上:** monolithic(単一の大きなヘッド)baseline と**タスク性能は同等**を保ちつつ、**意味のある expert 特化**を示す。データだけからモジュール的・解釈可能な方策が立ち上がる。

## 追っているプロジェクトへの適用可能性
- **P3(本命):** VLA / driving policy の action head を **MoE 化すると、行動の原始技能(車線維持・合流・停止など)が expert に自然に分離**しうる、という設計指針。手設計の階層なしで **解釈可能でモジュール的な方策**を得る道。デバッグ時に「どの expert が発火したか」で挙動を追える運用上の利点も大きい。
- **P3(効率):** MoE は疎活性なので、行動生成の計算を増やさず表現力を上げられる。on-vehicle 推論の計算配分の観点でも相性。
- **P1(横断):** expert 発火系列を「planner がどの技能モードにいるか」の可観測シグナルとして評価・監査に使える可能性。

### 試すべき実験
1. 我々の VLA/policy の action head を **monolithic → MoE** に差し替え、同パラメータ・同データで **タスク成功率が保たれるか**を確認(退行しないことがまず前提)。
2. **expert 発火とシナリオ種別(合流・右左折・停止等)の対応**を可視化し、原始技能への特化が driving でも創発するかを検証。再利用率(タスク横断での共有度)を測る。
3. 学習後の expert を固定し **新タスクへ router だけ再学習**して、原始技能の再利用による few-shot 適合(少データ適合)が効くかを試す。
