# Sample-Efficient Learning from Agent Experience（Experience Distillation）

**topic:** fm_distill_finetune (P2) ・ **arXiv:** 2607.21051v1 ・ **published:** 2026-07-23 ・ hf_upvotes: 7

## 何を解決する論文か
実世界の agent 学習は「環境との対話」が高コスト(実験を回す・人手フィードバックを取る)で、サンプル数がボトルネックになる。**in-context learning (ICL; 重みを更新せず、過去のやり取りを文脈に入れて学ぶ方式)** は少サンプルで賢くなれるが、その経験を文脈から外すと利得が消える。一方 **context distillation(文脈に入れていた情報を重みへ内在化する蒸留手法)**は重みに焼き付けられるが、agent の interaction history(試行錯誤の履歴)に、追加の環境対話なしで適用する方法が未開拓。本論文はこれを **Experience Distillation** と定式化し、収集済み経験だけで完結する実装を与える。

## 手法の核心
- **問題設定 Experience Distillation:** 「収集済みの agent 経験を、それ以上環境と対話せずに重みへ内在化する」タスクとして定義。環境サンプル効率を犠牲にしない点が肝。
- **ICL → 重みへの蒸留:** trial-and-error 経験を文脈に入れて ICL で得た振る舞いを、context distillation で重みに移す。文脈を外しても利得が残る。
- **強烈なサンプル効率:** 749 の software-engineering タスクと 6 つの text-adventure ゲームで、**ICL 利得の最低 64.8% を保持**。対して同じ経験に直接 SFT (Supervised Fine-Tuning; 収集した (入力,出力) をそのまま教師に微調整) しても **わずか 3.8% しか回収できない**。
- **RL との比較:** 「ICL で試行錯誤 → Experience Distillation」の組は、古典的な RL baseline と同性能を **9.6 倍以上少ない環境サンプル**で達成。

## 追っているプロジェクトへの適用可能性
- **P2(本命):** 「経験/文脈を重みに入れる」時、**素朴な SFT は経験の 96% を捨てている**という強い実測。我々の蒸留・適合パイプで「収集データを SFT で焼く」現状を、**context distillation 型に置き換えると同データからの回収率が跳ねる**可能性を示す、再現価値の高い recipe。
- **P1/P3:** planner や driving policy を「デモ/rollout 経験」から少サンプルで適合させる場面で、環境対話コストを 9.6 倍削れる筋。閉ループ実験が高コストな我々の設定に直結。

### 試すべき実験
1. 手持ちの小タスクで **同一経験に対し SFT vs Experience Distillation(ICL→context distillation)**を回し、**利得回収率**を実測。SFT が本当に一桁%しか拾えないかを自前で確認。
2. context に入れる経験量(履歴長)を振り、**内在化後に残る利得 vs 文脈長**の関係を測る。どれだけ長い試行錯誤まで焼けるかの上限を把握。
3. driving/planner の少数デモで、RL baseline と「ICL→蒸留」を**必要環境サンプル数**で比較。9.6 倍の効率が我々のタスクでも出るか。
