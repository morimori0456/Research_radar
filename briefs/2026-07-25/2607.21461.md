# AREX: Towards a Recursively Self-Improving Agent for Deep Research（wildcard）

**topic:** sns_wildcard (EXPLORE) ・ **arXiv:** 2607.21461 ・ **published:** 2026-07-22 ・ hf_upvotes: 115
**（枠の趣旨）** P1/P2/P3 の外だが注目度が高い探索枠。視野を広げる 1 件として採用。我々の research_loop 自体が「探索→検証→改善」の反復である点で、メタに学びが大きい。

## 何を解決する論文か
deep research(複数の制約を同時に満たす答えを探すタスク)では、**答えを見つけるのは高コストだが、候補を検証するのは制約ごとに分解すれば安い**——この **discovery–verification asymmetry(発見と検証の非対称性)**に着目。だから研究 agent は「ただ長く探す」のでなく、**中間結果を検証し、部分的に確定した状態を使って次の探索を導く**べき、と主張。AREX はこの **RSI (Recursively Self-Improving; 自分の出力を再帰的に検証・改善し続ける)** deep research agent の系。

## 手法の核心
- **二重ループ:** 証拠を集めて暫定解を作る **inner research loop** と、その解を**制約ごとに監査(constraint-wise audit)**し、未解決の主張を特定して**狙い撃ちの追加調査**を出す **outer self-improvement loop**。verify が cheap という非対称性を運用に落とす。
- **autonomous context-update tool を学習:** 長時間の反復で膨張する対話履歴を、**確定した証拠と未解決制約だけ残す compact な「改善状態」へ圧縮**するツールを、外部モデルに頼らず自前で学習。長ホライズンを維持する鍵。
- **学習法:** 検証済みの合成タスクと良質な軌跡で **agentic mid-training(本学習前の agent 適応段)**＋ **long-horizon RL(長い意思決定系列の強化学習)**。
- **sparse reward 対策:** 長ホライズンで最終報酬が疎になる問題に対し、**決定的な証拠を得た/誤った方向を修正した「key step」を強調**して密な学習信号にする。
- **実装:** dense な 4B モデルと 122B-A10B の **MoE (Mixture-of-Experts; 入力ごとに一部専門家だけ使う疎モデル)** を用意。BrowseComp / WideSearch / DeepSea(略)等で評価。

## 学び(なぜ視野を広げるか)
- **「探索より検証が安い」を設計原理に据える**発想は、我々の research_loop(候補選別→ブリーフ→実験)にもそのまま効く。**選別を長くするより、暫定選択を制約ごとに検証して弱い所だけ再探索**する運用への示唆。
- **history を「確定証拠＋未解決制約」に圧縮する context-update** は、長期タスクを回す agent の実務上のボトルネック(文脈膨張)への具体解。
- **long-horizon RL の sparse reward を key-step 強調で密にする**のは、driving policy や planner を長系列で RL する時(P1/P3)にも転用しうる一般的テクニック。

### 試すべき実験(メタ/転用)
1. 本パイプの候補選別を **「暫定 TOP-N を選ぶ → 各論文を relevance 制約ごとに再検証 → 弱い所だけ再探索」**の二重ループに変え、選別品質が上がるか小規模比較。
2. 長ホライズン RL を回す実験があれば、**key-step(決定的証拠/誤り修正)強調による reward shaping** を導入し、疎報酬下での学習安定性を測る。
