# HyWorldVLA: A Vision-Language-Action Model with Hybrid World Modeling for Autonomous Driving

**topic:** fm_distill_finetune (P2) ・ **arXiv:** 2607.20988v1 ・ **published:** 2026-07-23 ・ hf_upvotes: 0
**（注記）** candidates 上は P2 タグだが、内容は **P3(VLA + world model + E2E 自動運転)本命**、かつ **P1(NAVSIM での closed-loop 評価)にも直撃**。横断枠として採用。

## 何を解決する論文か
VLA (Vision-Language-Action; 画像と言語を入力に「行動」を直接出す基盤モデル) に world model(未来を予測するモデル)を足す E2E (end-to-end; 知覚から制御までを1本の学習可能なパイプラインで繋ぐ)自動運転は有望だが、未来予測の「粒度」にトレードオフがある。**pixel-level 予測**(未来のフレームをそのまま生成)は細かい時空間推論ができるがノイズに弱く、**latent 予測**(潜在空間だけで未来を表す)はロバストだが解釈性が落ち表現が劣化する。HyWorldVLA はこの二者を **1つの hybrid world model** に統合して両取りを狙う。

## 手法の核心
- **pixel と latent の二段構え:** pre-training では、pre-trained video VAE (Variational AutoEncoder; 動画を潜在ベクトルに圧縮/復元するモデル) が符号化した **video latent を予測**しつつ、同時に **video フレームを再構成**して pixel-level の grounding(潜在が実映像に紐づく状態)を与える。
- **co-fine-tuning で latent 専用へ:** 後段の co-fine-tuning(複数目的を同時に微調整するフェーズ)では **latent 特徴のみを予測**し、それを action expert(行動生成ヘッド)に渡して trajectory(軌道)を出す。推論時は重い pixel 生成を切り、latent だけで高速・頑健に動かす設計。
- **world model のノイズ頑健性を初めて定量評価:** NAVSIM v1/v2(自動運転の closed-loop planner ベンチマーク)で、pixel-based/latent-based の両 world model baseline を有意に上回る。
- **ノイズ環境での評価軸を新設:** noisy driving 下での world model の頑健性を定性・定量の両面で初めて包括分析し、「今後のアーキテクチャを測る新ベンチ」を提示。

## 追っているプロジェクトへの適用可能性
- **P3(本命):** 「world model の予測ターゲットを pixel か latent か」という設計判断に、**pre-train は pixel grounding 付き、実運用は latent 専用**という具体的な二段レシピを与える。我々が driving world model を組む時の default 構成の有力候補。
- **P1(横断):** NAVSIM 上で planner を world model rollout で評価する枠組みそのもの。**「pixel grounding を入れると noisy scene で planner 指標がどう動くか」**という評価視点は、我々の過保守/帰属曖昧さ論点に直結。
- **P2:** pre-train → co-fine-tuning の二相 adaptation は、基盤 world model を別ドメイン(自社シナリオ)へ適合させる recipe として転用可能。

### 試すべき実験
1. 我々の driving 設定で **latent-only vs pixel+latent(pre-train grounding あり)** を、clean と injected-noise(センサノイズ・遮蔽)の両条件で NAVSIM 系指標比較。pixel grounding が noisy 側でどれだけ効くかを切り分ける。
2. co-fine-tuning 段で pixel 予測を切るタイミングを数点振り、**pixel grounding の「効かせ量」と最終 planner 性能の関係**を測る(過剰な pixel 学習が表現劣化を招くか)。
3. action expert への latent 受け渡しを、我々の連続軌道 action 形式に合わせて置換し、trajectory smoothness(軌道の滑らかさ)と closed-loop 成功率で baseline と比較。
