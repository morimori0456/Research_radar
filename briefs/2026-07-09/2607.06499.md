# CE-MPPI: Clustering-Embedded Model Predictive Path Integral Control

- **arxiv-id**: 2607.06499 ／ **topic**: planner_ai ／ **project**: P1 ／ **relevance**: 3.5/5 ／ **hf_upvotes**: 0
- **URL**: http://arxiv.org/abs/2607.06499v1

## 何を解決する論文か
MPPI (Model Predictive Path Integral; 多数の軌道をランダムにサンプルし、コストで重み付け平均して制御入力を作る sampling-based な MPC の一種) は並列計算と相性が良く複雑な非線形系で強力だが、障害物が正面にある場面で「左回避の rollout 群」と「右回避の rollout 群」を重み付き平均してしまい、どちらでもない中間 (=障害物へ直進) に制御が潰れる **averaging-induced failure** を起こす。本論文はこの多峰性 (multimodality) の平均化問題を、クラスタリングを制御則に組み込むことで構造的に解決する。

## 手法の核心
- **軌道モードの分離**: サンプルした rollout を DBSCAN (Density-Based Spatial Clustering of Applications with Noise; 密度の濃い領域をクラスタとして見つけ、外れ値をノイズとして捨てるクラスタリング手法) でクラスタ化し、実行可能な軌道モードを不可能な rollout のノイズから分離する。
- **幾何学的 direction feature**: 衝突点由来の reference point から抽出した方向特徴をクラスタリングの距離尺度に使い、「左回避」と「右回避」のような幾何学的に非互換なモードを確実に分ける。
- **クラスタ選択ロジック**: 静的シーンでは最小コストのクラスタを選び、動的シーンでは obstacle flux (障害物の移動方向の流れ) と逆向きに舵を切るクラスタを選ぶ。重み付き平均は選ばれたクラスタ内でのみ行う。
- **実証**: 2D の JAX 加速シミュレーションで正面障害物での hesitation (躊躇による停滞) と移動障害物への持続的な巻き込まれを解消。実機 6-DoF UR5e マニピュレータ + Isaac Gym の CUDA 並列 rollout で time-to-goal 48% 短縮、end-effector 経路 12% 短縮。

## P1 (Planner AI + 評価) への適用可能性
- sampling-based planner の多峰性平均化は自動運転でも本質的に同じ問題 (対向車回避で左右の候補が拮抗する場面、cut-in への応答など)。MPPI 系や sampling-based trajectory optimizer を planner の一部に使うなら「クラスタ分離 → クラスタ内平均」は直接移植できる設計パターン。
- 評価の観点では、「rollout 群のクラスタ数・クラスタ間距離」は planner が多峰的な判断に迫られている度合いの指標になり、hesitation の検出メトリクスとして使える。
- 実機検証はマニピュレータであり自動運転の速度域・車両力学での検証はない点が relevance 3.5 の理由。

### 試すべき実験アイデア
1. **hesitation メトリクスの追加**: 自社 planner の closed-loop 評価に「候補軌道分布の多峰性 (クラスタ数) と最終出力のクラスタ所属」を記録し、averaging-induced failure に相当する事象 (どのモードにも属さない中間解) が実際に何 % 起きているか計測する。
2. **クラスタ内平均の ablation**: sampling-based な smoothing/fallback 層があるなら、DBSCAN + direction feature でモード分離してから平均する版と素の重み付き平均を、正面障害物・cut-in シナリオ群で closed-loop 比較する。
3. **動的シーンの選択則の検証**: 「obstacle flux と逆向きに選ぶ」ヒューリスティックが車両交通 (相手も反応する) で成立するか、インタラクティブなシミュレーションで確認する。
