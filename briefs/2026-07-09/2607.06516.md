# Point as Skeleton: Accumulated Point Cloud Enhanced Autoregressive Generation for Closed-Loop Autonomous Driving Simulation

- **arxiv-id**: 2607.06516 ／ **topic**: next_arch ／ **project**: P3 (P1 交差) ／ **relevance**: 5/5 ／ **hf_upvotes**: 0
- **URL**: http://arxiv.org/abs/2607.06516v1

## 何を解決する論文か
end-to-end 自動運転 (E2E-AD) の評価は、closed-loop の相互作用性 (CARLA 等のシミュレータ: ego の行動が環境に反映されるが見た目が合成的) と実世界の視覚忠実度 (nuScenes 等のログ再生: 映像は本物だが ego が逸脱すると破綻) のトレードオフを抱えてきた。本論文は、点群を「骨格」とした自己回帰的な走行映像生成で、**ego の逸脱に追従しつつ実写に近い映像を出す closed-loop な生成型センサシミュレーション**を実現する。

## 手法の核心
- **状態更新型の自己回帰生成**: ステップごとに更新される ego 状態・他車状態・地図・点群骨格を条件に、autoregressive な生成器が次の視覚観測を合成する。planner の出力が次フレームの条件に入るため真の closed-loop になる。
- **Reset-and-Roll**: rolling diffusion (映像を少しずつずらしながらデノイズする推論方式) をシミュレーションに適合させた推論法。未来条件に依存した潜在状態がシミュレーションステップを跨いで確定されるのを防ぎ、ステップ単位の状態更新と両立させる。
- **point-cloud skeleton**: 蓄積した点群から前景 (他車等のアセット) と背景を分離し、カメラ視点への painted-point (色付き点の投影) と template-depth (テンプレート深度) の2条件に射影して外観と幾何の手がかりを与える。自己回帰 rollout での誤差蓄積を安定化させる要。
- **nuPlan ベースの closed-loop 生成インターフェース**: ログからの ego 逸脱下で生成品質を評価できる renderer レベルの closed-loop 評価系を実装。nuScenes / nuPlan で closed-loop rollout 中の生成品質の向上を確認。コード公開 (github.com/krauwu/point-as-skeleton)。

## P3 (次世代アーキ) / P1 (評価) への適用可能性
- P3 の world model (予測・シミュレーション) と P1 の planner 評価 (closed-loop simulation) の交点に直撃する。E2E スタックを実写忠実度のある closed-loop で評価する基盤は、まさに P1 が求めるベンチマーク/評価手法の新提案でもある。
- 「生成型シミュレータの誤差蓄積を、点群という明示的な 3D 状態で係留する」設計は、純粋な video world model (画素のみで未来予測するモデル) の長時間破綻への現実的な対策として、P3 のアーキテクチャ設計に示唆がある。
- Reset-and-Roll の「未来条件の潜在を確定させない」工夫は、world model を planner-in-the-loop で使う際の一般的な落とし穴 (生成器が未来を先取りして因果が壊れる) への対処として単体でも学びがある。

### 試すべき実験アイデア
1. **公開コードでの closed-loop 評価試走**: nuPlan の閉ループインターフェースに自社 planner (または公開 E2E モデル) を接続し、ログ再生評価との指標乖離 (逸脱時の破綻頻度、衝突率の差) を測る。生成型評価がどの程度「使える」かの一次判断になる。
2. **誤差蓄積の骨格 ablation**: point-cloud skeleton 条件を外した rollout と比較し、何ステップで映像品質・幾何整合が破綻するかを計測する。自社で world model を作る際の「明示的 3D 係留の要否」の判断材料になる。
3. **評価用シナリオの増幅**: ログ中の危険シナリオで ego 行動を系統的に変えた counterfactual rollout (実際とは違う行動を取った場合の反実仮想展開) を生成し、planner の応答差を評価する回帰テストに使えるか検証する。
