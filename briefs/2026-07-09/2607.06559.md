# RynnWorld-4D: 4D Embodied World Models for Robotic Manipulation

- **arxiv-id**: 2607.06559 ／ **topic**: next_arch ／ **project**: P3 ／ **relevance**: 4.5/5 ／ **hf_upvotes**: 72
- **URL**: http://arxiv.org/abs/2607.06559v1

## 何を解決する論文か
ロボット操作の world model (行動や指示を条件に環境の未来を予測する生成モデル) は多くが RGB 画素のみを予測するため、幾何構造と運動の情報が行動生成から遠い。本論文は **RGB・depth・optical flow (画素ごとの動きベクトル) を同期した「RGB-DF」を単一の拡散プロセスで同時生成する 4D world model** を提案し、その内部表現から denoising を経ずに直接行動を出す policy まで一気通貫で示す。

## 手法の核心
- **RGB-DF 表現**: 見た目 (RGB)・幾何 (depth)・時間的運動 (optical flow) を同期生成することで、表現空間を低レベルの end-effector 行動に近づけ、world model と policy 学習のギャップを縮める、という設計仮説が中心。
- **tri-branch アーキテクチャ**: 3モダリティを別ブランチで持ちつつ cross-modal attention と frame-wise 3D RoPE (Rotary Position Embedding; 位置情報を回転行列で埋め込む方式の3D 拡張) で結合し、外観・幾何・運動が矛盾なく進化するよう制約する。入力は単一の RGB-D 画像 + 言語指示。
- **Rynn4DDataset 1.0**: egocentric な人間・ロボット操作映像 2.544 億フレームに、depth と optical flow の高品質 pseudo-label (既存モデルで自動付与した疑似教師) を付けた大規模データセットを構築。
- **RynnWorld-4D-Policy**: world model の内部 4D 表現を **1回の forward pass で消費する inverse dynamics head** (観測列から行動を逆算するヘッド) により、多段 denoising を回避して closed-loop で行動を出力。実機の両手 dexterous manipulation で SOTA、特に空間精度と時間協調が要るタスクで優位。

## P3 (次世代アーキ) への適用可能性
- 「world model の予測対象を RGB だけでなく幾何 (depth) と運動 (flow) に広げると、表現が行動に近づき policy が強くなる」という主張は、運転 world model の設計にそのまま検討価値がある。運転は depth (距離) と flow (相対運動) が意思決定のほぼ全てを占めるドメインであり、仮説の適合度はむしろ高い。
- 「生成のための多段 denoising」と「制御のための単一 forward」を分離し、内部表現だけを policy に流用する構成は、拡散系 world model を実時間の planning/制御に使う際のレイテンシ問題への実用的な解。
- pseudo-label で depth/flow を大規模付与するデータ戦略は、実センサの depth が無い/疎な走行映像データにも適用できる。manipulation 中心の検証で運転での実証がない点のみ留保 (relevance 4.5)。

### 試すべき実験アイデア
1. **予測ターゲットの ablation (運転版)**: 手元の運転 video 予測モデルで、RGB のみ vs RGB+depth+flow 同時予測を比較し、下流 (占有予測・planner 特徴としての線形評価) がどれだけ変わるか測る。pseudo-label は公開の depth/flow 推定モデルで付与する。
2. **内部表現からの planner head**: 学習済み world model の中間表現に軽量な行動/軌道ヘッドを付け、denoising を回さずに planning 特徴として使えるかを検証する (推論コストと性能のトレードオフを計測)。
3. **3D RoPE + cross-modal attention の移植**: マルチモダリティ (カメラ + LiDAR/レーダ由来 depth) を単一 backbone で扱う際の結合方式として、tri-branch + frame-wise 3D RoPE を自社アーキ候補と比較する。
