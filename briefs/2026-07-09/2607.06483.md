# Mitigating Domain Shift in Conditioned Floor Plan Generation: Synthetic Pre-training for Data-Efficient Adaptation

- **arxiv-id**: 2607.06483 ／ **topic**: fm_distill_finetune ／ **project**: P2 ／ **relevance**: 3.5/5 ／ **hf_upvotes**: 0
- **URL**: http://arxiv.org/abs/2607.06483v1

## 何を解決する論文か
条件付き間取り図 (floor plan) 生成モデルは、地域ごとの建築文化・空間制約の違いによる domain shift (訓練データと適用先のデータ分布のずれ) に極めて弱く、別データセットに転用すると性能が最大1桁劣化することを初めて系統的に示した。その対策として、**リアリズムを意図的に捨てて物理制約の遵守と多様性だけを保証した procedural な合成データ**での pre-training が、少データでの新ドメイン適合を大幅に改善することを示す。

## 手法の核心
- **domain shift の定量化**: 2つの異なる生成パラダイムの SOTA モデルを3つの公開データセット (RPLAN / MagicPlan / Swiss Dwellings) 間で交差評価し、最大1桁の性能劣化を確認。conditioned floor plan 生成でのこの検証は初。
- **合成データの設計思想**: procedural 生成 (ルールベースの手続きでデータを自動生成する方式) で、部屋の非重複・ドア配置の妥当性・グラフ整合性という**厳密な物理制約は強制**しつつ、極端に不規則な配置と激しい幾何摂動で**建築的リアリズムは意図的に犠牲**にする。domain randomization (見た目をランダム化して実ドメイン差をノイズとして学習させる手法) の生成モデル版。
- **効果 (zero-shot)**: この合成データでの pre-training だけで cross-domain の zero-shot 性能が大きく向上し、MagicPlan では in-domain 訓練を上回る。
- **効果 (fine-tuning 初期値)**: 新ドメインの少データ fine-tuning の初期値として、実データ初期値のベースラインを low-data regime で最大 40% 上回り、適合を加速する。

## P2 (FM 蒸留 + 適合) への適用可能性
- P2 の「少データでの適合・ドメインギャップの克服」に対する、そのまま使える一般 recipe: **「実データに似せる」のではなく「不変にしたい制約だけを守り、それ以外を極端にばらつかせた合成データ」で pre-training してから適合する**。他機種・他地域展開で target データが少ない状況にちょうど当てはまる。
- 「合成データはリアリズムが命」という直感への反証データ点として価値がある。守るべきは物理制約 (走行なら車両力学・衝突回避・地図整合性)、捨ててよいのは見た目の典型性、という切り分けの実例。
- ドメインは floor plan 生成で自動運転から遠く、モデル規模も小さい点が relevance 3.5 の理由。recipe の移植は自前の実験が必要。

### 試すべき実験アイデア
1. **制約優先の合成シナリオ pre-training**: 道路構造・車両力学・衝突制約だけを守り、地物配置や軌道形状を過剰にランダム化した procedural シナリオを生成し、新地域/新車種データでの fine-tuning 前の pre-training に使う。実データのみ初期値と low-data regime (例: target 数百件) で比較する。
2. **「リアリズムの犠牲」の効果分離**: 同じ procedural 生成で「リアルに寄せた版」と「不規則・摂動強めの版」を作り、どちらが cross-domain 転移に効くかを ablation する。本論文の主張がドメイン外でも成立するかの直接検証になる。
3. **domain shift の1桁劣化の追試**: 自社モデルを地域/車種の異なる評価データに zero-shot 転用したときの劣化幅をまず測る。劣化が小さいならこの recipe の優先度は下げてよい。
