# ABOPD: Antibody CDR Design via On-Policy Distillation

- **id:** 2607.18835v1 ｜ **topic:** fm_distill_finetune (P2) ｜ **hf_upvotes:** 0
- **URL:** http://arxiv.org/abs/2607.18835v1
- **authors:** Zhuo Yang, Jiaying He, Jiaqing Xie, Daolang Wang, Xipeng Qiu

## 何を解決する論文か
ドメインは抗体設計だが、本質は **on-policy distillation の recipe** 論文。生成モデルの標準的な denoising 訓練は「native 構造にノイズを足した状態」で学ぶが、実際の生成は「モデル自身が作った中間状態」を辿る。この **訓練分布と生成分布のミスマッチ** が、反復生成の途中で誤差を蓄積させ最終品質を崩す。ABOPD は、モデル自身の denoising 軌跡上の状態を、訓練時のみ使える privileged な native 幾何情報で教師することで、このズレを埋める。P2 が抱える「off-policy 蒸留だと student が自分の失敗分布を経験しない」問題への一般解。

## 手法の核心
- **on-policy distillation:** teacher(privileged な native geometry)が、student **自身の denoising 軌跡が実際に訪れる状態** を監督する。off-policy(固定データ上の蒸留)や supervised fine-tuning より、生成時に遭遇する誤差状態を直接矯正できる。
- **privileged information の活用:** native 構造という「推論時には使えない特権情報」を訓練時のみ teacher 側に与える(privileged learning; teacher だけが正解付近を見る設定)。
- 反復生成(denoising trajectory)に沿った fine-grained な構造監督により、誤差蓄積を抑える。
- 結果: RAbD の CDR-H3 生成で RMSD(構造の平均二乗誤差)を 2.37Å→1.95Å(−0.42Å)改善し、supervised fine-tuning と offline distillation を上回る。

## P1/P2/P3 への適用可能性と実験アイデア
- **P2(FM 蒸留 + 適合)本命の recipe。** 「student 自身の生成軌跡上で teacher が監督する」on-policy distillation は、反復/自己回帰生成する全てのモデル(diffusion policy、world model、autoregressive planner)に横展開できる。off-policy 蒸留が抱える exposure bias(自分の生成を経験しないズレ)への処方箋。
- **横断で [[2607.19191]](ABot-World-0)と同じ悩み。** ABot-World-0 の LongForcing は「student の長 self-rollout を teacher に整合」= 本質的に on-policy distillation の long-horizon 版。**両者を並べて読むと「生成分布上で teacher を当てる」共通原理が見える。**
- **実験1:** 我々の重い teacher → 軽量 student の蒸留を、(a) 固定データ上の offline 蒸留 vs (b) student の生成軌跡上での on-policy 蒸留 で比較し、closed-loop の drift / 最終品質の差を測る。
- **実験2:** privileged information の AD 版として「訓練時のみ使える ground-truth 未来軌跡 / HD map」を teacher に与え、推論時 map-less な student を on-policy 蒸留 → domain gap 下での頑健性を評価。
