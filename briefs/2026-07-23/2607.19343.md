# Masked Visual Actions for Unified World Modeling

- **id:** 2607.19343v1 ｜ **topic:** next_arch (P3) ｜ **hf_upvotes:** 3
- **URL:** http://arxiv.org/abs/2607.19343v1
- **authors:** Hadi Alzayer, Wenlong Huang, Haonan Chen, Christopher Luey, Lvmin Zhang, ほか

## 何を解決する論文か
video model(動画生成モデル)は「物がどう動き・接触し・反応するか」の物理的 prior を大量に学んでいるが、そこに **action(行動)をどう入力するか** が難しい。テキストや制御ヒントで action を渡すと、モデルが学んだ視覚空間とズレる。本論文は action を「動画中の任意物体の *部分的に開示された軌跡*(pixel 空間の軌跡)」として表現する **Masked Visual Actions** を提案し、1 つの world model(環境の未来を予測するモデル)を forward dynamics(行動→次状態の予測)にも inverse(望む結果→必要な行動)にも使えるようにする。

## 手法の核心
- **action = pixel 空間の部分開示軌跡。** ロボット自身の動きを開示すれば「低レベル行動→シーンの応答」を予測する forward dynamics model として振る舞い、望む物体の動きを開示すれば「その結果と整合するロボット挙動」を復元する inverse model になる。同一 checkpoint で両方向。
- action を視覚空間に埋め込むので、video model が学んだ interaction prior とネイティブに整合する(別モダリティで action を注入しない)。
- 実写+シミュの masked example を **わずか 15 時間** で finetune するだけで、多様なシーン・複数 embodiment(機体)にまたがる視覚的忠実度と可制御性を達成。
- 下流の manipulation で、(a) imagined rollout(想像上の展開)の結果が実行結果と相関 → **policy evaluation に使える**、(b) 候補となる複数の未来を順位付けして model-based planning の意思決定を改善、(c) 望む物体運動からロボット運動を合成(inverse)。

## P1/P2/P3 への適用可能性と実験アイデア
- **P3(次世代アーキ)本命。** 「action を視覚空間の軌跡として与える」設計は、自動運転の world model にほぼ直移植できる着想。BEV(Bird's-Eye-View; 俯瞰図表現)や pixel 空間で「自車/他車の未来軌跡を部分開示 → シーン応答を予測」する条件付けは、テキスト条件より AD の action 空間(連続的な軌跡)と相性が良い。
- **P1(planner 評価)を横断で射抜く。** 「imagined rollout の結果が実行と相関 → 候補未来のランキングで planning 改善」は、まさに我々が NAVSIM で回している closed-loop 評価の代替/補完になりうる。**実験:** 学習済み driving world model に自車候補軌跡を Masked Visual Action として与え、生成 rollout 上で collision / progress を測って planner をランキング → 実 closed-loop メトリクスとの相関を検証(open-loop で closed-loop を近似できるか)。
- **少データ finetune(15h)** の効率は P2 の適合観点でも参考。既存の video / world model を AD ドメインへ「軌跡開示 finetune」で安く適合できるかを ablate する価値。
