# Point Ladder Tuning: Parameter-Efficient Hierarchical Adaptation for 3D Point Cloud Understanding

- **id:** 2607.19171v1 ｜ **topic:** fm_distill_finetune (P2) ｜ **hf_upvotes:** 0
- **URL:** http://arxiv.org/abs/2607.19171v1
- **authors:** Junlin Chang, Longhao Zou, Rui Li

## 何を解決する論文か
事前学習済み point-cloud バックボーン(LiDAR 等の 3D 点群を処理するモデル)の適合は通常全パラメータを更新し、計算・メモリが重い。さらに現代の point backbone は aggressive な tokenization/downsampling(積極的にトークン化・間引き)で「コンパクトな global token」を作る一方、細かい局所幾何を不可逆に捨てる。このため既存の **PEFT**(Parameter-Efficient Fine-Tuning; 少パラメータだけ更新する適合)は粗い token 上でしか働かず、失われた multi-scale の局所性を回復できない。本論文はバックボーンを凍結したまま、局所性を意識した階層的・instance 条件付き適合を行う **Point Ladder Tuning (PLT)** を提案する。

## 手法の核心
- **バックボーンは凍結**し、軽量な閉ループを外付けする(PEFT)。
- **Hierarchical Ladder Network (HLN):** 生の点群から直接、多解像度の局所特徴ピラミッドを構築 — downsampling で捨てられた局所幾何を別経路で拾い直す。
- **Local-Global Fusion (LGF):** 局所ピラミッドと、凍結バックボーンの中間 semantics を整列・融合。
- **Dynamic Prompt Generator:** instance ごとの multi-scale prompt を生成し、凍結バックボーンを効果的に変調(prompt tuning; 入力側に学習可能な手がかりを差し込む適合)。
- dense prediction 用に、融合特徴を段階的に upsample する軽量 segmentation head を追加。
- 結果: 分類で **学習パラメータ 2.71%**、dense prediction で 7.69% で従来 PEFT を上回り、PointGPT-L では **0.36%** で大型バックボーンにスケール。コード公開。

## P1/P2/P3 への適用可能性と実験アイデア
- **P2(FM 蒸留 + 適合)、AD 直結。** 自動運転の知覚は LiDAR 点群が中核で、事前学習済み point backbone を各車種・各センサ構成へ安く適合したい要求と一致。**「downsampling で捨てた局所幾何を ladder で拾い直す」** という診断は、小物体・遠距離点の検出が効く AD で特に価値がある。
- **実験1:** 我々の点群バックボーンを凍結し PLT で 3D 検出/セグメンテーションに適合 → full fine-tune との精度/パラメータ Pareto を比較(特に小物体 recall)。
- **実験2:** センサ構成違い(別 LiDAR 機種)への **domain adaptation** を、少データ + PLT の instance-aware prompt だけで実現できるか。前日の DA-MergeLoRA 的な「ドメイン別軽量モジュール」思想と組み合わせる余地。
- **横断:** 「凍結特徴 + 極小 head で full fine-tune を超える」構図は、直近の Patch Policy(07-22 [[2607.18236]])と同じ *capacity 設計* の系譜。凍結表現の価値を測る比較実験の対象に加える。
