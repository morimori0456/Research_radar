# DA-MergeLoRA: Hypernetwork-Based LoRA Merging for Few-Shot Test-Time Domain Adaptation

**topic:** fm_distill_finetune (P2) ／ **arXiv:** 2607.17467v1 ／ **hf_upvotes:** 0

## 何を解決する論文か
FSTT-DA (Few-shot Test-Time Domain Adaptation; 新ドメインへ、少数のラベルなしサンプルだけを使い推論時に適合する設定)は、訓練時に target データへアクセスできる通常の domain adaptation より現実的だが、既存手法は浅い BatchNorm 更新・モデルを黒箱扱いする prompt・cross-domain 関係を捉えないアンサンブルに頼り、source ドメイン固有の知識を活かせていない。本論文は **LoRA fine-tuning と model merging を統合**した FSTT-DA 枠組みを提案する。

## 手法の核心
- **LoRA (Low-Rank Adaptation; 重み行列に低ランクの差分だけを学習する PEFT = Parameter-Efficient Fine-Tuning 手法)** モジュールを、CLIP の vision encoder 上で source ドメインごとに個別 fine-tune。LoRA は少パラメータしか触らないため、base の汎化を保ちつつドメイン固有特徴を内部に学ぶ。
- **hypernetwork(別ネットワークの重みを生成するネットワーク)** を meta-learning で訓練し、source LoRA 群を混ぜるための **per-column merging factor** を生成。
- target の少数ラベルなし画像バッチを与えると、hypernetwork が merging weight を出し、source LoRA 群を1つの適合表現へ融合。
- 複数の DA データセットで SOTA。code 公開。

## P1/P2/P3 への適用可能性と実験アイデア
- **P2(高優先の "他ドメインへの適合" 直撃):** 「ドメインごとの LoRA ライブラリ + 学習された merging」という再利用可能な recipe。少データ・**無ラベル**での適合という P2 の高優先要件をそのまま満たす。**実験:** 運用ドメイン(夜間・雨天・都市・センサ構成)ごとに LoRA adapter を訓練し、新規デプロイ先の少数フレームから hypernetwork で融合 adapter を生成。single-domain LoRA / 一様平均マージに対する優位を測る。
- **capacity/gap の観点:** source LoRA を何個まで増やすと merging が飽和するか、というスケーリング問いも検証価値あり。
- **留保:** ベースは CLIP の分類系タスク。planner/回帰系への転用は merging factor の粒度(per-column)がそのまま効くか要検証。
