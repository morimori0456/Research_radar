# Patch Policy: Efficient Embodied Control via Dense Visual Representations

**topic:** fm_distill_finetune (P2) ／ **arXiv:** 2607.18236v1 ／ **hf_upvotes:** 0

## 何を解決する論文か
事前学習された ViT (Vision Transformer; 画像をパッチに割って transformer で処理する視覚バックボーン) の**密なパッチ特徴**は強力なのに、robot policy ではほとんど活かされていない。既存 policy は各観測を単一の global token に潰して空間の細部を捨てるか、逆に billion-param 級の VLM (Vision-Language Model) バックボーンを丸ごと担いで重く・遅くなるかの二択だった。Patch Policy は、軽量な transformer policy が**事前学習済みの密なパッチ token を直接消費**できる最小限の構造拡張を提案する。

## 手法の核心
- frozen(凍結)な ViT の**per-patch 密特徴**をそのまま入力にし、global pooling で1 token に潰さない。
- **block-causal attention mask**: 標準的な policy の時間的因果性(未来を見ない)を保ちつつ、1 観測あたり多数のパッチ token + 状態情報に attention を張れるようにするマスク設計。
- full VLM バックボーンを持たないので軽量・高速。
- 結果: 4 つの sim + 3 つの実機スイートで、SOTA な global-pooled 表現に対し **+40% 相対改善**。さらに **fine-tune 済み OpenVLA-OFT を +18% 上回りつつ、使用パラメータは約 0.7%**。

## P1/P2/P3 への適用可能性と実験アイデア
- **P2(本日の capacity gap の教訓):** 「大きな VLA の価値の大半は *frozen 特徴* の中にあり、バックボーン全体を fine-tune しなくても、良い密特徴 + 極小 head で取り出せる」。これは capacity gap(teacher と student の表現力差が開きすぎて蒸留が失敗する現象)への実務的な回答で、student を巨大にせずとも良い frozen 特徴を消費させれば足りる可能性を示す。**実験:** 蒸留/適合ターゲットに対し (a) 大バックボーンの full fine-tune vs (b) frozen 密特徴 + 軽量 block-causal head を、param・latency・精度の Pareto で比較。
- **P3:** block-causal mask over patch tokens は、空間の細部を保ったまま高頻度で反応する**コンパクト driving policy** のアーキ recipe。**実験:** 運転向けに事前学習した ViT の frozen 密特徴 + 小 policy head が、同一計算量下で E2E fine-tune モデルを超えるか検証。
