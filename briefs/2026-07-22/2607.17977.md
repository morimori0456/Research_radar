# RynnBrain 1.1: Towards More Capable and Generalizable Embodied Foundation Model

**topic:** next_arch (P3) ／ **arXiv:** 2607.17977v1 ／ **hf_upvotes:** 31(本枠で本日最高の注目度)

## 何を解決する論文か
embodied(身体を持つエージェント向け)foundation model の family を 2B / 9B / 122B-A10B の 3 スケールで提示。embodied perception・spatial reasoning・localization・planning を統一的に扱い、v1.0 に比べて表現と出力を robot manipulation により直接整合させることを狙う。

## 手法の核心
- 統一的な spatio-temporal かつ物理接地(physically grounded)された訓練枠組み。
- family 全体に **contact-point prediction(接触点予測)** を追加、2B/9B には **native 3D grounding** を追加し、出力を manipulation に寄せる。
- **RynnBrain-VLA**: **cross-embodiment action space(異なるロボット機体で共通の行動空間)+ embodiment-specific masking(機体ごとに一部だけ有効化するマスク)** を持つ VLA。Unitree G1・Astribot-S1・Tianji-Wuji に実機デプロイ。
- **122B-A10B**(A10B = MoE = Mixture-of-Experts; 総パラメータを増やしつつ一部の専門家サブネットのみ活性化して計算量を抑える構造。ここでは活性パラメータ約 10B)が、VSI-Bench・MMSI・RefSpatial-Bench で proprietary・open 両方を上回る。
- 実機で RynnBrain 初期化 policy が Qwen ベース/汎用 VLA を上回り、multi-task・multi-embodiment の joint 訓練が per-task 訓練を上回る。

## P1/P2/P3 への適用可能性と実験アイデア
- **P3:** embodied/VLA foundation model を**スケールさせる際の設計参照**。特に「cross-embodiment action space + embodiment-specific masking」は、1つのバックボーンを異種プラットフォーム(車種・アクチュエータ・センサ構成違い)に跨って訓練する recipe。**実験:** 複数の運転プラットフォーム(操舵系が異なる)に対し「共通 action space + embodiment mask」を採り、共有バックボーン事前学習 → 機体別 head を訓練し、multi-embodiment joint vs per-platform を比較。
- **P2:** MoE 122B-A10B は「総容量を増やしつつ推論コストを抑える」実例で、capacity/効率の観点で示唆的。**注目度が高く(hf=31)標準ベースライン化する公算が大きいため、P2 の蒸留 teacher 候補として追跡価値**。
- **留保:** ベンチは spatial reasoning 系。自動運転タスクへの転移は未検証で、あくまで設計思想と teacher 候補としての価値。
