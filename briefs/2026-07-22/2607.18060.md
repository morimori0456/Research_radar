# RoboHarness: Memory-Driven Orchestration of Heterogeneous Robot Policies for Long-Horizon Planning

**topic:** planner_ai (P1) ／ **arXiv:** 2607.18060v1 ／ **hf_upvotes:** 0

## 何を解決する論文か
long-horizon なタスクは単一 policy では賄えない多様な能力を要求するが、既存の planner は「適用範囲が固定された同質のスキル集合」を前提に組まれており、異種 policy を束ねる際の**能力境界の不確かさ**と **cross-policy の分布ミスマッチ**を無視している。RoboHarness は、独立に開発された制御システム(VLA・RL policy・TAMP など)を再利用可能な "agentic skill" として統一的にカプセル化し、能力境界を推定しながら routing する枠組みを提案する。

## 手法の核心
- 独立開発の制御系(VLA、RL policy、TAMP = Task-and-Motion Planning; 記号的タスク計画と連続的な動作計画を同時に解く古典的枠組み)を、統一インターフェースの再利用可能スキルとして包む。
- **multi-modal execution memory + online evidence** で各 policy の capability boundary(その policy が信頼できる状態領域)を特徴づけ、capability-aware な分解と routing を行う。
- **Memory Bridge**: policy の受け渡し(handoff)を安定化させる中核機構。次に使う policy の実行 trajectory を retrieve し、その policy の in-distribution state region(訓練分布内の状態領域)を推定して、ロボットをそこへ誘導する。**joint retraining 不要**。
- 3 つの公開ベンチ + 500 の自作タスク + 135 の実機実験で、zero-shot long-horizon planning と OOD (Out-Of-Distribution; 訓練分布外) robustness の改善を確認。

## P1/P2/P3 への適用可能性と実験アイデア
- **P1(本命の持ち帰り):** 自動運転スタック自体が異種 planner/policy の集合(rule-based、learned、MPC)である。「各 planner の in-distribution state region を推定し、それを *評価シグナル* にする」発想が使える。**実験:** シナリオごとに候補 planner の in-distribution 推定を記録し、planner 間の不一致(disagreement)を「どの planner も自信のない領域」の検知=fallback trigger / eval 指標として使う。
- Memory Bridge の「次 policy の in-distribution 領域へ能動的に寄せてから渡す」は、**learned planner → safety fallback controller** の安全な handoff 設計にそのまま転用できる着想。
- **P3:** VLA + 古典モジュールを合成する meta-architecture の参考。
- **留保:** 実験ドメインは manipulation(把持・組立系)で自動運転ではない。移せるのは「能力境界の推定と routing」という設計思想レベルで、具体手法の直接移植ではない。
