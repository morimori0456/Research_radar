# LaMem-VLA: VLA モデルの latent 空間ネイティブな Dual Memory

- **arxiv-id**: 2607.07608 ／ **topic**: next_arch ／ **project**: P3 ／ **relevance**: 4.5/5 ／ **hf_upvotes**: 44
- **URL**: http://arxiv.org/abs/2607.07608v1

## 何を解決する論文か
主流の VLA (Vision-Language-Action; 画像と言語指示を入力に行動を直接出力する基盤モデル) は現在の観測だけから行動を予測する Markovian な仮定 (現在の状態だけで次が決まるという仮定) に立つため、長時間・時間依存のタスクに弱い。既存のメモリ拡張 VLA は観測窓の拡大や履歴検索を「ポリシーの外側の補助文脈」として足すだけで、履歴が VLA の推論と同じ latent 空間で混ざり合わない。LaMem-VLA は履歴を latent memory token に再構成し、VLA の推論系列に直接編み込む **latent-memory-native** な枠組みを提案する。

## 手法の核心
- **curator (整理役)**: 履歴経験を short-term と long-term の 2 つの相補的な memory vault (記憶庫) に整理する。直近の文脈と長期のタスク進行を分けて持つ dual 構成。
- **seeker (検索役)**: 現在のマルチモーダル認識をクエリに両 vault を検索し、文脈に関連する証拠だけを取り出す。全履歴を注意対象にしない点で計算が有界。
- **condenser (圧縮役)**: 検索された証拠をコンパクトな short-term / long-term の latent memory token に再構成する。
- **weaver (編込み役)**: memory token を現在の観測・指示と 1 本の連続 embedding 系列として注入する。履歴の表現・検索・消費がすべて同一の連続 latent 空間で完結するため、記憶が multimodal 推論と行動生成に直接参加できる。
- SimplerEnv・LIBERO (いずれもロボット操作の標準シミュレーションベンチマーク) で既存手法を上回る。

## P3 (次世代アーキテクチャ) への適用可能性
- 自動運転の E2E モデル/VLA も「現在フレーム + 短い履歴」の Markovian 設計が主流で、長時間の文脈 (数分前に見た標識、通過済みの工事区間、対向車との過去のインタラクション) を落としている。**メモリを外付けモジュールでなく同一 latent 系列に編み込む**という設計指針は運転ドメインにそのまま持ち込める。
- bounded context で長期履歴を扱う構成 (検索 + 圧縮で token 数を抑える) は、車載の計算・レイテンシ制約と相性が良い。
- 同日候補の NativeMEM (2607.06678; 1 フレーム 1 token に圧縮する軽量アプローチ) と設計思想が対照的で、「リッチな検索型 dual memory vs 極限圧縮の native memory」として読み比べる価値がある。

### 試すべき実験アイデア
1. **memory-demanding な運転タスクの切り出し**: 自社データから「過去の観測を覚えていないと正解できない」シナリオ (一時的に隠れた歩行者、過去の標識に依存する制約) を抽出し、現行モデルの失敗率を測って long-horizon メモリの必要性をまず定量化する。
2. **latent memory token の最小実装**: 既存 E2E/VLA モデルの入力系列に、過去フレームを凍結 encoder で圧縮した数 token を追加するだけの簡易版を作り、上記シナリオ群で効果を見る (weaver 相当のみの ablation)。
3. **short/long dual 構成の検証**: 直近数秒 (short-term) と走行開始からのイベント要約 (long-term) を分けて持たせる構成と、単一メモリの構成を比較し、dual 化の寄与を確認する。
