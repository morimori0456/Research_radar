# Behavior Leverage Imbalance: Multi-Teacher On-Policy Distillation の挙動シフトと Soft Clamp

- **arxiv-id**: 2607.07050 ／ **topic**: fm_distill_finetune ／ **project**: P2 ／ **relevance**: 4.5/5 ／ **hf_upvotes**: 0
- **URL**: http://arxiv.org/abs/2607.07050v1

## 何を解決する論文か
tool を呼ぶべき場面と直接回答すべき場面を持つ agentic な LLM では、「tool 呼び出し担当の teacher」と「直接回答担当の teacher」から multi-teacher の on-policy distillation (OPD; 学生モデル自身の生成分布上で teacher と一致させる蒸留。off-policy な教師データ模倣と対比される) で学習するのが自然だが、素朴にやると**集計 loss には現れない挙動シフト** — 直接回答すべき場面でも tool を呼ぶ over-calling — が起きることを示した。原因を token レベルで特定し、per-token の発散を校正する **Soft Clamp** で修正する。

## 手法の核心
- **現象**: 2 teacher 構成の GKD (Generalized Knowledge Distillation; on-policy サンプルに対して teacher との分布間発散を最小化する蒸留の一般形) では tool-call の recall は上がるが、over-calling も 13.7% 発生。しかも集計指標では説明がつかない — tool-call サンプルが token 露出で多いわけでも、tool-call teacher の系列全体の per-token divergence が大きいわけでもない。
- **behavior leverage imbalance という診断**: `<tool_call>` タグや関数名などの mode-entry / 構造 token における局所的な token レベルの信号が、生成全体のモード (tool を呼ぶか直接答えるか) に不釣り合いに大きな支配力を持つ。「teacher 信号の大きさの集計」ではなく「**どこに**効いているか」を見ないと蒸留の副作用は見えない。
- **Soft Clamp**: token ごとの Jensen-Shannon divergence (JSD; 2 分布間の対称な発散尺度) が極端な値を取る token について、発散を動的に圧縮しつつ勾配は非ゼロに保つ per-token の校正法。ハードに切り捨てず「soft に潰す」ことで学習信号を殺さない。
- **結果**: APIGen-MT で over-calling を 13.7% → 9.0% に削減しつつ decision accuracy は vanilla GKD と同等。BFCL (Berkeley Function Calling Leaderboard; tool 呼び出し能力のベンチマーク) の multi-turn 診断でも tool-call のループや重複呼び出しが減少。

## P2 (FM 蒸留 + 適合) への適用可能性
- relevance_criteria の「新しい loss」「再現可能な recipe」に直球。multi-teacher 蒸留・複数スキルの統合蒸留を回すなら、「集計 loss が健全でも挙動分布が歪む」という failure mode とその診断手順 (token 位置別の divergence 分析) はそのまま実務チェックリストになる。
- 教訓は tool-use に限らない。蒸留で「モード選択」(例: 慎重な出力 vs 積極的な出力、書式 A vs 書式 B) を持つタスクなら、mode-entry token への leverage 集中は普遍的に起こりうる。
- 実装コストが低い (loss の per-token 重み付けの変更のみ) のも実務向き。

### 試すべき実験アイデア
1. **蒸留パイプラインへの診断の組み込み**: 現行の蒸留学習に「token 位置別 JSD の分布ログ」を追加し、構造 token (特殊タグ・区切り・モード切替 token) に発散が集中していないかを可視化する。まず診断だけでも価値がある。
2. **Soft Clamp の追試**: 自社の teacher-student ペアで vanilla KD loss と Soft Clamp 版を比較し、出力の挙動分布 (モード選択の比率) が teacher とどれだけ一致するかを精度と別軸で計測する。
3. **multi-teacher 構成の検証**: 単一 teacher で能力が拮抗しない場合に「スキル別 teacher + Soft Clamp」で統合する構成を小規模に試す (例: ドメイン特化 teacher + 汎用 teacher)。
