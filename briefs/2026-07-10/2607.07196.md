# Validate the Dream Before You Trust Its Verdict: World Model シミュレータの admissibility 認定

- **arxiv-id**: 2607.07196 ／ **topic**: next_arch ／ **project**: P3 (P1 評価にも直結) ／ **relevance**: 4.5/5 ／ **hf_upvotes**: 0
- **URL**: http://arxiv.org/abs/2607.07196v1

## 何を解決する論文か
world model (行動の帰結を想像上の世界で予測するシミュレータとして使える生成モデル) をポリシー評価の test oracle (合否判定器) に使う流れが加速しているが、その判定 (verdict) は world model 自体が信頼できて初めて意味を持つ。動画生成型 world model の評価は FVD (Fréchet Video Distance; 生成動画と実動画の分布距離で視覚的リアリズムを測る指標) など見た目の忠実さに偏っており、「ポリシーの行動に世界が正しく応答するか」を検証していない。本論文は、world model を評価器として使う前に認定する **admissibility ladder (L0–L4)** を提案し、自動運転で実証する。

## 手法の核心
- **問題の転倒を指摘**: 古典的なシミュレーションベース検証は「信頼済みシミュレータで未検証ポリシーを評価する」構図だが、生成的 world model は**シミュレータ自体が未検証の学習物**。判定を証拠として使うには、まず world model を accredit (認定) しなければならない。
- **安全工学の credibility 実務を輸入**: VV&A (Verification, Validation & Accreditation; モデルが正しく作られ・現実を写し・用途に対し公式に認定されているかの体系)、SOTIF (Safety of the Intended Functionality; 故障ではなく機能の仕様不足に起因するリスクを扱う自動車安全規格 ISO 21448)、シナリオベーステスト標準の考え方を world model 認定に接続する。
- **admissibility ladder L0–L4**: L0 (視覚的生成品質) から始まり、行動追従性 (action-following; 入力した行動どおりに世界が応答するか。学習分布外の行動を含む) などの段階を登り切って初めて closed-loop の verdict を assurance evidence (安全性の論拠) として採用できる、という段階的な関門を定義。embodiment 非依存の枠組み。
- **AD での実証で逆転現象**: 2 つの driving world model に適用すると、視覚的生成品質 (L0) で上位のモデルが行動追従性 (L1–L2) では下位という**逆転**が起きた。つまり FVD 的な見た目の良さは、closed-loop 判定が依存する action-robustness を予測しない。

## P3 (次世代アーキテクチャ) への適用可能性
- world model を開発するなら、この ladder は「何を満たせば評価・学習に使ってよいか」の受け入れ基準の叩き台になる。特に「見た目指標と行動追従性は乖離する」という実証結果は、world model の学習目標・中間評価の設計を見直す根拠になる。

## P1 (Planner AI + 評価) への適用可能性
- relevance_criteria の「評価手法の新提案」に合致。planner 評価に学習ベースのシミュレータ/world model を混ぜる場合、その判定の信頼性をどう担保するかはまさに評価基盤の問題。L1–L2 (行動追従性) のテストプロトコルは自社評価パイプラインの health check として輸入できる。

### 試すべき実験アイデア
1. **行動追従性テストの実装**: 手持ちの driving world model (または学習ベース sim) に対し、同一初期状態から系統的に異なる行動系列 (急ブレーキ、車線変更、分布外の操舵) を入力し、世界の応答が行動と整合するかを自動判定するテストスイートを作る — L1–L2 の最小実装。
2. **視覚品質と行動追従の相関測定**: 自社/公開の world model 複数で FVD 系指標と行動追従スコアを並べ、論文の「逆転」が自社設定でも再現するか確認する。再現するなら学習の目的関数に行動条件付けの一貫性項を足す動機になる。
3. **verdict の信頼度ゲート**: world model ベースの合否判定を planner 評価に使う際、ladder のどの段まで通過したかで verdict の重み (人手レビューへの回付基準) を変える運用を設計する。
