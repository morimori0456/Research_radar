# When Does Knowledge Distillation Hurt? Reliability-Aware Distillation for Low-Resource Language Summarization

**topic:** fm_distill_finetune (P2) ・ **arXiv:** 2607.19956v1 ・ **published:** 2026-07-22 ・ hf_upvotes: 0

## 何を解決する論文か
Knowledge distillation (KD; 大きな teacher モデルの出力分布を小さな student に写して圧縮する手法) は「平均すれば効く」前提で使われるが、**サンプル単位で見ると多くの例はむしろ student を悪化させている**ことを本論文は実測する。Bangla 要約ベンチ BanSum で、標準 KD は ROUGE-L を +0.0003 しか上げず、しかも訓練サンプルの約 **51.3% が student の validation loss を実際に悪化させている**と推定した。そこで「どのサンプルで蒸留すべきか」を選ぶ reliability-aware な蒸留を提案する。

## 手法の核心
- **CHAD (Counterfactual Harm-Aware Distillation):** 各サンプルの KD の有用性を、その勾配が validation loss を下げる方向とどれだけ揃うか(gradient alignment)で測る。この counterfactual な判定を全訓練集合に一般化する軽量な gate(蒸留する/しないを決める小さな分類器)を学習。
- **EWAD+CPDP:** token-level entropy-weighted adaptive distillation(token ごとの entropy = 予測の不確かさで蒸留の重みを変える)に、2つ目の(語彙非互換な)teacher からの **capacity-proportional geometric constraint** を組み合わせる。これが capacity gap(teacher と student の能力差が大きすぎて素直に真似られない問題)への直接の対策。
- **結果:** 標準 KD が +0.0003 のところ、CHAD は +0.0173、EWAD+CPDP は +0.0219 ROUGE-L。わずか 60M パラメータの student が、50倍大きい fine-tuned Qwen2.5-3B を上回る。
- **一般性:** EWAD+CPDP を XL-Sum の 15 言語で検証し 10/15 でベースライン超え。2つの teacher が相補的な信号を出す所で効き、双方が飽和/弱い所で効かない、と効き所も切り分け。
- コードと学習済みモデルを公開(selective distillation の再現用)。

## 追っているプロジェクトへの適用可能性
- **P2(本命):** 我々の蒸留 recipe は「全データで一様に KD」しているが、本論文の主張が正しければ**半分近いデータが逆効果**。CHAD の gradient-alignment gate は「どのサンプルを蒸留に使うか」を原理的に選別する即戦力で、再現可能な recipe。CPDP の capacity-proportional constraint は、基盤モデル(大)→ 実務用小モデルの capacity gap 対策として我々の設定に直結。
- **P3:** world model / VLA を蒸留で小型化する際にも「有害サンプル除外」は横断的に効く。

### 試すべき実験
1. 現行の蒸留パイプラインで **per-sample KD 有用性**(student の validation loss 方向と KD 勾配のコサイン)を計測し、「有害サンプル比率」を出す。50% 近ければ本論文の主張が我々のドメインでも成立。
2. gradient-alignment gate で有害サンプルを除外して再蒸留 → 一様 KD と精度/収束速度を比較。安ければ CHAD の gate を軽量 MLP で実装し、閾値を掃引。
3. teacher を2つ(例: 大 teacher + 別系統の中 teacher)用意し、EWAD+CPDP の capacity-proportional constraint を入れて capacity gap の大きい student での改善幅を測る。
