# Unified Prediction and Planning via Conflict-Aware Disjoint Parameter Training

**topic:** planner_ai (P1) ・ **arXiv:** 2607.19971v1 ・ **published:** 2026-07-22 ・ hf_upvotes: 0

## 何を解決する論文か
周辺エージェントの motion prediction(将来軌跡の予測)と自車の safe planning は密結合だが、edge デバイスに載せるにはこの2タスクを1つの compact な共有 encoder に統合したい。ところが「他者の挙動を予測する」目的と「自車中心で安全に計画する」目的は表現レベルで衝突し、共有パラメータを奪い合って両方とも中途半端になる。本論文はこの **Skill Conflict**(異なるタスクが同じ重みを取り合い、どちらにも特化しきれない現象)を同定し、モデルマージで解く。

## 手法の核心
- **Skill Conflict の特定:** unified モデルで prediction 用と planning 用のパラメータ割り当てが重複し、勾配が競合する領域を明示的に切り分ける対象として定義。
- **Disjoint Parameter Training (DPT):** 各タスクの「核となるパラメータ領域」を分散学習で分離し、能力を保ったまま後で1モデルに merge する model-merging ベースの枠組み。
- **Sparse merging:** 全タスク固有パラメータを混ぜるのではなく、各タスクで最も影響の大きいパラメータだけを選んで統合。隣接特徴どうしの干渉を防ぎ、表現容量を集中させることで最良性能。
- **手法非依存:** DPT は既存の各種 merging 手法(task arithmetic 等)と並行して適用できる。
- crowd navigation ベンチ **JRDB / JTA** で、compact な共有モデルながら prediction・planning 双方で優位。

## 追っているプロジェクトへの適用可能性
- **P1(本命):** 我々の AD planner でも「他車 prediction」と「ego planning」を1つの backbone に載せると同じ Skill Conflict が起きているはず。DPT の「タスク別に核パラメータを分離 → sparse merge」は、multi-head を単に足す現状より原理的。特に「片方のタスクを足すと他方が劣化する」を定量観測する診断としてすぐ使える。
- **P2:** merging / sparse selection は distillation 後の複数 student 統合や、機種別 fine-tune の後統合にも転用余地。ただし本命は P1。

### 試すべき実験
1. 現行の unified prediction+planning モデルで、prediction loss と planning loss の勾配を層ごとに取り、**勾配コサイン類似度が負(=競合)になる層/チャネル**を可視化 → Skill Conflict の有無を実測。
2. prediction 専用・planning 専用に別々に軽く学習 → **sparse merge(影響度上位の重みだけ統合)** で1モデル化し、full-share baseline と NAVSIM 等の closed-loop 指標で比較。merge 率(top-k%)を掃引して collision/progress のトレードオフ曲線を出す。
