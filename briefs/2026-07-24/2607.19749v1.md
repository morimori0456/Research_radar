# The World Model Remembers, the Actor Forgets: Dream Rehearsal for Continual Model-Based RL

**topic:** next_arch (P3) ・ **arXiv:** 2607.19749v1 ・ **published:** 2026-07-22 ・ hf_upvotes: 0

## 何を解決する論文か
DreamerV3 系の model-based RL エージェント(world model = 環境のダイナミクスを潜在空間で予測するモデルを学習し、その「夢」の中で policy を学ぶ)は、タスクを順番に学ぶと catastrophic forgetting(以前のタスクを破滅的に忘れる現象)を起こす。過去経験を全部保持する replay buffer があっても忘れる。本論文は continual-RL 分野が答えを仮定してきた「**どの構成要素が忘れているのか**」を初めて実測し、忘却は「記憶の問題」ではなく「channel(出力経路)の問題」だと示す。

## 手法の核心
- **成分別プローブ(事前登録・n=3 seed):** never-clear replay 下で、world model は旧タスクについて測れるものをほぼ全て保持(reward 弁別の retention ratio ≈ 1.0、value 推定、termination 構造)。一方 **actor の振る舞いだけが崩壊**する。
- **介入で因果を示す:** world model を凍結し同一の imagined rollout を与えても、想像内での RL では失った skill を回復できない(0/3 seed)。ところが **world model 自身の「採点済みの夢」への supervised self-imitation** なら、環境と一切対話せず 3/3 seed で回復。
- **Graded dream rehearsal:** これを訓練中に interleave すると、タスクラベル不要・パラメータ数一定の continual learner に。plain replay が 0/3 の 4タスク連鎖を 3/3 保持、8タスク連鎖も 3/3、実エピソード cloning より一貫して上(paired diff +0.13、bootstrap 95% CI [0.07, 0.24])。
- **dream 採点が肝(load-bearing):** scoring の2つの失敗モードを特徴づけ、offline の選別ゲージで事前検出、realized-first grading rule で塞ぐ。全実験が事前登録で、棄却された仮説も報告。

## 追っているプロジェクトへの適用可能性
- **P3(本命):** driving world model を複数シナリオ/ドメインで継続学習する時、「world model は覚えているが policy 側が忘れる」という切り分けは設計指針として強い。つまり忘却対策のリソースは replay 容量でなく **policy 更新経路(dream rehearsal)** に振るべき、という具体的な結論。
- **P1:** 「学習済み world model は旧環境の reward/termination を保持している」という測定結果は、planner を world model の rollout で評価する際に「モデル側の忘却」を疑わなくてよい根拠になり、評価の信頼性を上げる。

### 試すべき実験
1. 我々の world model + policy を2〜3シナリオ順に学習し、**成分別 retention プローブ**(world model の reward/value 保持 vs policy 成功率)を取る。actor だけ崩れる現象が driving でも出るか確認。
2. graded dream rehearsal(world model の夢を採点して self-imitation)を interleave し、plain replay と旧シナリオ保持率を比較。環境対話ゼロで回復するかが鍵。
3. dream 採点の failure mode(過大/過小評価)を offline gauge で監視する仕組みを、我々のシミュ指標に載せて再現。
