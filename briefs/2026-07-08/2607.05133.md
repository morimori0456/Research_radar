# UNIVERSE: Unified Video Action Models for Autonomous Driving with Flexible Mask-Modulated Modality Generation

- **arxiv-id**: 2607.05133 ／ **topic**: fm_distill_finetune ／ **project**: P2 (P3 に強く交差) ／ **relevance**: 4/5 ／ **hf_upvotes**: 0
- **URL**: http://arxiv.org/abs/2607.05133v1

※ fetch は fine-tuning キーワードで P2 に分類したが、内容は P3 (world model / end-to-end 自動運転) の中核論文。P2 観点 (転移・汎化) でも高得点のため P2 枠で採用し、両プロジェクト向けに書く。

## 何を解決する論文か
WAM (World Action Model; 将来の video 予測を dense supervision に使って行動学習を強化する枠組み) を自動運転に使うとき、既存の cascaded / dual-DiT 設計 (video 生成と action 予測を別モジュールに分ける構成) では、video で学んだ世界のダイナミクスが trajectory 側に間接的にしか伝わらず、action 側がデータセット固有の運転バイアスに過学習する。本論文は **単一の DiT (Diffusion Transformer; 拡散モデルを transformer で構成した生成アーキテクチャ) に video と action を統合**し、この伝達ロスを解消する。

## 手法の核心
- **単一 mask-modulated DiT**: 将来 video の latent と ego-trajectory (自車軌跡) のトークンを **同じ生成パラメータで co-training**。video 予測の dense supervision が trajectory の denoising を直接形作る。
- **Modality-Decoupling Visibility Mask**: 過去の文脈 (履歴) は両モダリティで共有しつつ、**未来 video トークンと未来 trajectory トークンの相互 attention を遮断**。未来の正解情報が漏れる future-target leakage を防ぎ、因果的妥当性を保つ。
- **推論時は trajectory-only**: マスク設計のおかげで、テスト時は未来 video の denoising を丸ごと外して trajectory だけ生成できる。joint rollout 比で **4.3x 高速化**、planning 精度は同等。video-only / joint 生成も同一モデルで可能。
- **結果**: NAVSIM (記録データ上で planner を評価する planning ベンチマーク) で **91.0 PDMS** (PDMS = Predictive Driver Model Score; 衝突・進行度などを合成した総合運転スコア。Two-DiT 構成は 89.6)。**nuScenes と Bench2Drive へ fine-tuning なしで zero-shot 転移**。ablation で単一 DiT 統合・video co-training・mask の三点がそれぞれ効くことを確認。

## P2 / P3 への適用可能性
- **P3 (次世代アーキテクチャ)**: 「world model と E2E planner を別モジュールにするか統合するか」という設計論争に対し、単一 backbone + attention mask による統合が cascaded 構成を上回る具体的な証拠。次世代アーキ検討の比較基準 (baseline) に据える価値がある。
- **P2 (適合・転移)**: video co-training が **dataset-specific な運転バイアスへの過学習を抑える正則化**として働き、zero-shot でのドメイン転移 (nuScenes / Bench2Drive) を実証している点は「基盤モデルを別ドメインへ転用する recipe」として直接参考になる。転移先データが少ない状況で video 予測タスクを補助にする、という手が示唆される。

### 試すべき実験アイデア
1. **trajectory head の video co-training 正則化**: 自社 planner の学習に将来 video (または BEV occupancy 系列) 予測を共有パラメータの補助タスクとして追加し、別データセット/別地域への zero-shot 転移性能が上がるか検証する。
2. **visibility mask の再現**: 未来モダリティ間の attention 遮断あり/なしで leakage と closed-loop 性能を比較し、mask 設計の効き方を自社設定で確認する。
3. **推論時モダリティ脱着の検証**: 学習時 joint・推論時 trajectory-only の速度/精度トレードオフを測り、車載リアルタイム制約下での実用性を見積もる。
