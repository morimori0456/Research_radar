# GaP: A Graph-as-Policy Multi-Agent Self-Learning Harness For Variational Automation Tasks

- **arxiv-id**: 2607.05369 ／ **topic**: planner_ai ／ **project**: P1 ／ **relevance**: 3/5 ／ **hf_upvotes**: 0
- **URL**: http://arxiv.org/abs/2607.05369v1

## 何を解決する論文か
産業・商用ロボットで求められる「Variational Automation」(固定自動化より物体の形状・姿勢のばらつきが大きいタスク群) では、model-free policy (環境モデルを持たず観測から行動を直接学習する方策) は信頼性が足りず、手書きプログラムは適応性が足りない。本論文は、LLM ベースの multi-agent coding システムに **解釈可能な計算グラフをポリシーとして生成・自己改良させる** ことで、プログラムの信頼性と学習ベースの適応性を両立させる。

## 手法の核心
- **Graph-as-Policy (GaP)**: perception / planning / control のノードからなる directed computation graph を、MORSL (Modular Open Robot Skill Library; 再利用可能なロボットスキル部品集) を組み合わせて multi-agent coding harness が自動生成する。ポリシーが「コードとグラフ」なので人間が読んで検査できる。
- **内部 simulation の自動生成**: タスクごとに rehearsal 用のシミュレーション環境を生成し、異なるグラフ候補を並列に試行 (rehearse) する。TAMP (Task and Motion Planning; 記号的なタスク計画と幾何学的な動作計画を統合する古典枠組み) と ROS の発想を LLM エージェントで再構成した形。
- **反復改良ループ**: rehearsal の成功率とスループットを目的関数に、グラフの構造とパラメータを反復的に改良する。学習が「重みの更新」ではなく「プログラムの編集」で進む。
- **評価**: 新設の 8 つの Variational Automation ベンチマーク (シミュレーション4 + 実機4) でベースラインを大きく上回る成功率。コード・データ公開。

## P1 (Planner AI + 評価) への適用可能性
- 直接の対象は manipulation/automation であり自動運転プランナーそのものではない (relevance 3 の理由)。ただし「**解釈可能なプランナー表現 + 自動生成された closed-loop simulation での rehearsal + 反復改良**」というループ構造は、P1 のプランナー開発・評価基盤にそのまま写せる設計思想。
- 特に「タスクインスタンスを内部 sim として自動生成し、複数の候補構成を並列評価する」部分は、プランナーのパラメータチューニングや回帰テストの自動化に示唆がある。
- 「model-free では信頼性ギャップが埋まらない領域では、構造化 (グラフ/コード) されたポリシー + 自動改良が勝つ」という主張は、E2E 一辺倒への反証データ点としても記録価値あり。

### 試すべき実験アイデア
1. **失敗シナリオ → 回帰テスト自動生成**: 実走行/シミュレーションで planner が失敗したシナリオから、GaP 式に「rehearsal 用タスクインスタンス」を自動生成し、planner 変更のたびに並列 closed-loop 評価する回帰基盤を試作する。
2. **パラメータ自動改良ループ**: 自社 planner の設定パラメータ (コスト重み等) を、LLM エージェントに closed-loop 指標を見せながら反復編集させ、人手チューニングとの到達点・工数を比較する。
3. **グラフ構造の diff 監査**: 改良ループが加えた変更を人間がレビューできる形式 (グラフ diff) で残し、「自動改良の説明可能性」が評価プロセスとして成立するか検証する。
