# InternVLA-A1.5: Unifying Understanding, Latent Foresight, and Action for Compositional Generalization

- **arxiv-id**: 2607.04988 ／ **topic**: next_arch ／ **project**: P3 ／ **relevance**: 5/5 ／ **hf_upvotes**: 18 (本日候補中の本枠最高)
- **URL**: http://arxiv.org/abs/2607.04988v1

## 何を解決する論文か
ロボット操作の unified モデル (VLM の意味理解と、将来予測で学ぶ物理ダイナミクスを1つの方策に統合する路線) は、実際には (1) pretrained VLM backbone の意味理解を訓練中に壊す、(2) 異種目的 (理解・予測・行動) が干渉する、(3) 将来予測を pixel 空間でゼロから学び、pretrained video 生成モデルが持つダイナミクスの事前知識を活かせない、という三重苦を抱える。本論文はこの三つを同時に解く構成を提案する。

## 手法の核心
- **VLM semantics の保持**: policy を native VLM backbone の上に建て、行動学習中も VQA (Visual Question Answering; 画像への質問応答タスク) と subtask 予測の訓練を継続することで、pretrained の意味理解を侵食させない。行動生成は軽量な unified expert (行動専用の付加モジュール) が担う。
- **latent-querying としての将来予測**: 将来予測を pixel 生成ではなく **latent への問い合わせ**に再定式化。少数の学習可能な **foresight tokens** が「タスクに関係する未来」を圧縮 latent code に凝縮する。
- **frozen video 生成モデルからの蒸留**: foresight tokens の学習は **凍結した pretrained video generation model の supervision** 下で行う。つまり world model のダイナミクス事前知識を、pixel 生成を一切学ばずに継承する。**video branch は推論時に破棄**されるため、リアルタイム制御が保たれる。
- **結果**: 1.2M robot episodes + 3M multimodal samples で事前学習し、6 つのシミュレーションベンチマークすべてで最高成績。実機では、保持された semantics により未見の instruction 組み合わせに対する compositional generalization (部品の新しい組み合わせへの汎化) が最強で、長 horizon 実行も持続。

## P3 (次世代アーキテクチャ) への適用可能性
- P3 の中心テーマ「VLA と world model をどう統合するか」への現時点で最も洗練された回答の一つ。「**world model を推論時に走らせず、frozen video 生成モデルから latent 蒸留で事前知識だけもらう**」という設計は、車載のリアルタイム制約と world model の恩恵を両立させたい自動運転にそのまま示唆がある。
- 「backbone に元タスク (VQA 等) を混流し続けて意味理解の劣化を防ぐ」処方は、P2 の catastrophic forgetting (適合時に元能力を忘れる現象) 対策とも同型で、P2/P3 横断の recipe。

### 試すべき実験アイデア
1. **foresight token の移植**: 自社 VLA/planner prototype に少数の foresight tokens を追加し、既製の video diffusion model (走行 video で fine-tune 済みならなお良い) を frozen teacher として latent 蒸留する。推論コストを増やさずに将来予測の恩恵が出るかを検証。
2. **意味理解の混流訓練**: 行動学習中に運転ドメインの VQA / シーン記述タスクを混ぜ、instruction の新規組み合わせ (未見の指示×場所) への汎化が向上するか測る。
3. **蒸留先の比較**: 「pixel 予測を直接学ぶ」「latent foresight を蒸留する」「予測なし」の3構成を同一データで比較し、UNIVERSE (2607.05133) の統合型 co-training とどちらが自社設定に合うかを見極める。
