# TGRIP: A Text-Guided Approach to Vehicle Instance Prediction in Autonomous Driving

- **arxiv-id**: 2607.04812 ／ **topic**: fm_distill_finetune ／ **project**: P2 ／ **relevance**: 4/5 ／ **hf_upvotes**: 0
- **URL**: http://arxiv.org/abs/2607.04812v1

## 何を解決する論文か
BEV (Bird's-Eye View; 上空から見下ろした俯瞰座標系) での end-to-end instance prediction (物体単位の将来予測) は、occupancy 回帰や optical flow といった **幾何学的な教師信号だけ**で学習されており、エージェントを「動く障害物」としか扱えない。追い越しや交差点のように物体の意味 (何であるか・どう振る舞う種類か) が将来予測を左右する場面で曖昧さを解けない。本論文は Vision-Language Foundation Model の意味的知識を教師信号として注入し、この意味の欠落を埋める。

## 手法の核心
- **teacher-student pipeline**: 大規模な Vision-Language Foundation Model (teacher) がマルチカメラ画像から **semantic-enhanced BEV maps** (意味情報を埋め込んだ俯瞰マップ) を生成し、これを student ネットワークの学習中の auxiliary supervision (補助的な教師信号) として使う。
- **蒸留は学習時のみ**: teacher は訓練時の教師信号生成にだけ使われ、推論時は student 単体で動く。つまり車載計算コストを増やさずに foundation model の意味的知識だけを移植する、annotation 不要の dense supervision。
- 学習される spatio-temporal 表現は「幾何学的に整合しつつ意味的に識別的」になり、semantic guidance を将来 instance prediction という時系列タスクに統合した初の試みと主張。
- **結果**: nuScenes で既存 SOTA を上回る。コード公開。

## P2 (Foundation Model 蒸留 + 適合) への適用可能性
- P2 の本題である「**大きな foundation model の知識を車載サイズのネットワークへ蒸留する recipe**」に自社ドメイン (自動運転知覚・予測) で直撃する。しかも蒸留の媒体が feature でも logits でもなく **タスク空間の中間表現 (semantic BEV map)** である点が実務的に扱いやすい: teacher と student のアーキテクチャが違っても成立し、capacity gap の影響を受けにくい。
- 「幾何タスクに意味を足す」という枠組みは instance prediction に限らず、planner の入力表現や occupancy 予測にも横展開できる。

### 試すべき実験アイデア
1. **semantic BEV auxiliary loss の追加**: 自社の prediction/planner スタックに、既製 VLM (Vision-Language Model) で生成した semantic BEV map を auxiliary loss として追加し、意味依存シナリオ (交差点・割り込み) での予測精度変化を測る。
2. **教師信号の生成コスト評価**: VLM による BEV map 生成をオフラインの一括前処理として回した場合のコスト/スループットを見積もり、「人手 annotation の代替」としての損益分岐を出す。
3. **teacher 品質の感度分析**: teacher VLM のサイズ・種類を変えて student 性能への感度を測り、「どこまで安い teacher で済むか」の蒸留 recipe として整備する。
