# Efficient PEFT Methods with Adaptive Checkpointing for Vision Models and VLMs on Resource Constrained Consumer-GPUs

- **arxiv-id**: 2607.02158v1 ／ **topic**: fm_distill_finetune ／ **project**: P2 ／ **relevance**: 4/5
- **URL**: http://arxiv.org/abs/2607.02158v1

## 何を解決する論文か
事前学習済み視覚モデルは高精度だが微調整に大きな GPU メモリを要し、エッジ展開が非現実的になる。本論文は **2GB 級の VRAM 予算下**で 5 種の PEFT 手法(Full FT / LoRA / AdaLoRA / QLoRA / BitFit)を Transformer 系(ViT-Small, TinyViT)と Mamba 系(Vim-Small, MambaVision-T)バックボーンで横断比較し、さらに**メモリ予算対応の適応的 gradient checkpointing** を提案する、実務直結の recipe 研究。

## 手法の核心
- **統一プロトコル比較**: 精度・学習時間・エネルギー・NetScore 系(多目的指標、展開志向の2変種を追加)で PEFT×バックボーンを CIFAR-100 / DTD 上で公平比較。
- **適応 checkpointing アルゴリズム**: メモリ予算を守りつつ再計算箇所を動的選択。ピークメモリを **43–79% 削減**(エネルギー 9–30% 増のトレードオフ)。
- **効率の勘所**: **QLoRA と BitFit はエネルギーを 20–30% 削減し精度低下は 1–2%** に留まる。
- **基盤モデル比較**: zero-shot 対照系(OpenCLIP, SigLIP)、自己教師(DINOv2 + 軽量評価)、自己回帰 VLM(PaliGemma, MobileVLM, SmolVLM)を対照。
- **意外な結果**: **DINOv2 は微調整モデルを CIFAR-100 で上回り(0.917 vs 0.897)**、しかも桁違いに省エネ。小型自己回帰 VLM はまだ非競争的。

## P2(FM 蒸留 + 適合)への適用可能性
- 「他機種・他ドメインへの適合を**限られた VRAM で回す**」という P2 の核心そのもの。特に本ループの実行環境(Jetson 級エッジ)で微調整/オンデバイス適合を検討する際の意思決定表になる。
- 適応 checkpointing は、蒸留後の生徒モデルをエッジで追加微調整する際のメモリ律速を外す実用ツール。
- 「強い自己教師 backbone(DINOv2)を凍結して軽量ヘッドだけ学習」の方が微調整より省エネで高精度、という知見は蒸留/適合パイプラインの前提設計を変えうる。

### 試すべき実験アイデア
1. 自社ドメインデータで QLoRA vs BitFit vs 凍結DINOv2+線形ヘッドを、(精度, ピークメモリ, エネルギー)のパレートで比較し、エッジ適合のデフォルト recipe を確定。
2. Mamba 系バックボーン(MambaVision-T)を PEFT した場合の精度/メモリ挙動を Transformer 系と比較 — SSM 系の適合効率を P3 の関心とも接続。
3. 適応 checkpointing を自社の蒸留後追加微調整に組み込み、2GB 予算で回せる最大バッチ/解像度を実測。
