# RoboTTT: Context Scaling for Robot Policies

**arXiv:** 2607.15275 | **topic:** next_arch (P3) | **score:** 4.5/5 | **hf_upvotes:** 12

## 何を解決する論文か

近年の robot foundation model は単一ステップか短い履歴の visuomotor context しか使わない。RoboTTT は **TTT (Test-Time Training; 推論時にも勾配降下でモデル重みを更新する仕組み)** を robot foundation model に組み込み、visuomotor context を **8K timestep** (従来比 3 桁) までスケールしつつ推論レイテンシを増やさない学習レシピを提案する。この長 context で、人間動画からの one-shot in-context 模倣、実行中のポリシー改善、摂動への頑健性、多段・長期タスクの性能向上が解放される。

## 手法の核心

- **fast weights による recurrent state**: TTT を VLA (Vision-Language-Action) ポリシーに統合し、履歴を**重み空間に圧縮**する系列モデルにする。recurrent state = 学習・推論の両方で勾配降下により更新されるパラメータ (fast weights) で、長 context の情報検索を担う
- context 長を増やしても推論レイテンシが伸びない (state 圧縮なので O(1) 的更新。attention の長系列コストを回避)
- **学習レシピ**: sequence action forcing (行動系列を段階的に条件付けする教師強制) と truncated backpropagation through time (BPTT を有限長で打ち切る勾配計算) を組み合わせ、長 context の学習を可能にする
- 実ロボット操作で単一ステップ baseline 比 **+87%**、5分・10段の組立を完遂 (baseline は一度も不可)。8K context 版は 1K context 版比 **+62%**
- **「context 長が新しい scaling 軸」**: 事前学習の context 長を伸ばすと閉ループ性能が単調改善する現象を初めて観測

## P3 への適用可能性と実験アイデア

- P3 の中核テーマ「新しい構造・学習効率・scaling 則」に直撃。運転 E2E/VLA で history をどう扱うかは未解決で、**「長 context を attention でなく fast weights に圧縮」**という設計は、可変長の走行履歴 (数分の運転文脈) を低レイテンシで扱う候補になる
- closed-loop 性能が context 長で単調に伸びるという観測は、運転でも「事前学習 context 長」を scaling 軸に加える動機づけになる
- 実験: 運転 policy に TTT recurrent state を足し、履歴長 (1s→数十s) を振って NAVSIM/閉ループ指標がどこで飽和するかを測る。特に occlusion 後の再認識や、信号待ち→発進のような長期依存タスクで fast-weights 圧縮が KV-cache 方式より遅延/精度でどう勝るかを A/B
