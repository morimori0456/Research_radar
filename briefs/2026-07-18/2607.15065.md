# DriftWorld: Fast World Modeling through Drifting

**arXiv:** 2607.15065 | **topic:** next_arch (P3) | **score:** 4/5 | **hf_upvotes:** 0

## 何を解決する論文か

world model (行動の結果を「想像」して計画に使う予測モデル) は、制御に効くかどうかが「rollout をどれだけ速く大量に出せるか」に懸かる。diffusion ベースの world model は多ステップ sampling のため 1 rollout が高価で、推論時の大規模 action search がボトルネックになる。DriftWorld は **drifting generative model (推論時に反復 denoising せず、学習時に学んだ drift で一発生成する生成方式)** に基づく action-conditioned world model で、この速度問題を解く。

## 手法の核心

- **action-conditioned drift を学習時に獲得**: 推論では反復 denoising せず、現在の観測と候補 action 系列から**単一 forward pass** で未来フレームを生成 (30+ fps、diffusion baseline 比平均 **17倍高速**)
- 速さと精度を両立するので、推論時に多数の candidate action 系列を並列 rollout する **action search** が実用域に入る
- 標準の vision-based 操作ベンチ (Bridge-V2, RT-1, Language Table, Push-T, Robomimic) で、diffusion world model 系より遥かに少ない推論時間で SOTA の意思決定性能
- **オフライン simulator としても機能**: 実ロボット policy のランキング用途で、rollout ベースのスコアが ground truth と最大 **0.99** 相関 (= 実機評価の代理指標になり得る)

## P3 への適用可能性と実験アイデア

- P3 の world model 軸に直結。特に「**速い imagination が planning と policy 評価の両方を支える**」という主張は、運転の (a) 推論時 action search と (b) オフラインでのプランナー評価の双方に効く。後者は P1 の評価パイプラインとも橋渡しになる
- diffusion world model の 17倍高速化を「一発生成の drift」で得る設計は、運転シーンの多数候補軌道を rollout してコスト評価する用途に輸入候補
- 実験: 運転ログで action-conditioned DriftWorld を学習し、(1) 候補軌道集合を rollout してコスト最小を選ぶ closed-loop 制御、(2) 実機/シム policy のランキング (rollout スコア vs 実測 EPDMS の相関) を測る。特に 30fps 級で「1 制御周期あたり何本の candidate を評価できるか」を実測し、DRIFT 型の proposal 生成と組み合わせられるか検討
