# SEED: Self-Evolving On-Policy Distillation for Agentic Reinforcement Learning

**arXiv:** 2607.14777 | **topic:** fm_distill_finetune (P2) | **score:** 4.5/5 | **hf_upvotes:** 64

## 何を解決する論文か

LLM を長期タスクのエージェント (multi-turn 対話・tool 使用・環境フィードバック) として学習する際、outcome-based RL (最終結果だけで報酬を与える強化学習) は報酬が trajectory 単位で疎で、途中の判断への手がかりが乏しい。SEED は、完了した on-policy 軌道を「hindsight skill (後から振り返って抽出した再利用可能な手順・教訓)」に変換し、その効果を dense な token 単位の distillation 信号としてポリシーに蒸留し返すことで、この監督ギャップを埋める。

## 手法の核心

- **hindsight skill 抽出**: ポリシー自身を fine-tune して、完了軌道を分析し「再利用可能な workflow・決定的な観測・失敗回避ルール」を自然言語 skill として生成させる
- **self-evolving ループ**: RL 中、現在のポリシーが軌道を収集すると同時に analyzer も兼ねて skill を抽出する。ポリシー更新が「意思決定」と「skill 分析」を同時に改善するため、hindsight 監督がポリシーと共に進化する (別途固定の teacher を要しない — capacity gap 対策の観点で新しい)
- **skill-induced probability shift の蒸留**: サンプルした行動を「通常 context」と「skill を足した context」の両方で再スコアし、その確率差を **dense な token 単位 on-policy distillation 信号**に変換
- この蒸留信号を outcome-based RL と**同時最適化**するので、補助監督が常に現在の軌道分布と整合したまま保たれる (off-policy 蒸留の分布ずれを回避)
- text ベース・vision ベース双方の agentic タスクで性能と sample efficiency を一貫して改善、未知シナリオへの汎化も確認。コード公開 (github.com/jinyangwu/SEED)

## P2 への適用可能性と実験アイデア

- P2 の distillation レシピとして「teacher を別に用意せず、生徒自身の on-policy 軌道から dense 信号を作る」という点が新規。固定 teacher が用意できない/capacity gap が大きい社内設定で、**self-distillation で dense 化する**発想が輸入できる
- 実験: 社内の RLVR (Reinforcement Learning with Verifiable Rewards; 検証可能な報酬での RL) パイプラインに SEED の「skill 抽出→確率差を蒸留信号化」段を追加し、疎報酬タスクで sample efficiency (同一性能到達までの環境ステップ数) を比較
- ablation として、skill を「自然言語」ではなく構造化スロット (workflow/観測/回避ルールの固定フィールド) にした場合の probability shift の安定性を測り、蒸留信号の分散低減に効くか検証
