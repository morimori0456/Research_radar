# ABot-N1: Toward a General Visual Language Navigation Foundation Model

**arXiv:** 2607.10383 | **topic:** sns_wildcard (EXPLORE) | **score:** 4/5 | **hf_upvotes:** 84

(昨日の wildcard 枠で Direct-OPD に僅差で譲った候補。本日も候補に再掲されており、注目度・学びの価値が高いため本日の探索枠として採用)

## 何を解決する論文か

VLN (Visual Language Navigation; 言語指示と視覚観測から移動を行うタスク) の基盤モデルは、観測から行動への monolithic な直接写像だと座標のドリフトや long-tail な意味理解に弱く、中身も説明できない。ABot-N1 は **slow-fast アーキテクチャ** (低速で深い推論と高速な制御を分離する2系統構成) により、汎用性・頑健性・解釈可能性を同時に狙う。

## 手法の核心

- **slow 系**: vision-language reasoner が CoT (Chain-of-Thought; 途中の推論過程を明示的に生成させる方式) で明示的に推論し、**pixel goal (画像空間上のアンカー点)** を出力する
- この画像空間アンカーが**タスク横断の統一インタフェース**になる: point-goal / object-goal / POI-goal / 指示追従 / 人物追従をすべて同じ形式で扱える
- **fast 系**: action expert がテキストの手がかりと pixel goal の両方を受けて、制御周波数で連続 waypoint を生成する
- 高レベル意図と低レベル制御を「pixel 接地されたアンカー + 明示的な言語トレース」で橋渡しするため、判断根拠を後から追える
- シミュレーションと実世界ベンチマークの両方で SOTA

## P3 への学び (分野外だが)

- 運転でも「VLM で深く推論する slow 系 + 軽量な軌道生成の fast 系」という dual-system 構成は主要な設計候補であり、本論文はその**インタフェース設計の具体例**として価値が高い。数値 waypoint や言語でなく「画像空間のアンカー点」を仲介にすると、座標ドリフトに強く、複数タスクを1つの出力形式に畳める、という主張は運転 VLA にそのまま輸入できる仮説
- 実験アイデア: 運転 VLA で高レベル module の出力を (1) テキスト指示、(2) BEV 座標の goal、(3) カメラ画像上の pixel goal、の3形式で比較し、下流の軌道生成の頑健性 (特にカメラキャリブレーションずれや未知環境) を測る
