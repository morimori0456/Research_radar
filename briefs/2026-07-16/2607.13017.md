# FlowWAM: Optical Flow as a Unified Action Representation for World Action Models

**arXiv:** 2607.13017v1 | **topic:** next_arch (P3) | **score:** 4/5

## 何を解決する論文か

WAM (World Action Model; 事前学習済み動画生成器を world model と policy の両方に流用する枠組み) では「action をどう表現すれば動画生成器と噛み合うか」が未解決だった。数値ベクトルの action は動画生成器の表現形式と揃わず、既存の視覚的 action 表現はフレーム間の時間的な動き構造を捨てている。FlowWAM は **optical flow (フレーム間の各ピクセルの移動ベクトル場) を action 表現そのものにする** ことでこれを解く。

## 手法の核心

- flow の動画は RGB 動画と同じ形式なので、**同一の事前学習動画生成器で RGB と flow を dual-stream (2系統を並行して生成する) diffusion として同時にモデル化**できる
- 1つのモデルが2モードで動く: policy mode では flow を生成して action 予測に使い、world-model mode では目標の flow 系列を条件に未来の動画を生成する
- flow は生動画から自動抽出できるため、**action ラベルなしの大規模動画で事前学習可能**
- RoboTwin (ロボット操作ベンチマーク) で成功率 92.94% (Clean) / 92.14% (Random) と VLA (Vision-Language-Action; 視覚・言語入力から行動を出力するモデル) や既存 WAM を上回り、WorldArena の world model 評価でも軌道精度を相対 18.4% 改善

## P3 への適用可能性・実験アイデア

- 運転ドメインは「action ラベルなし動画 (フリート走行映像・公開ドラレコ) が大量、ラベル付きが少量」という構造なので、**flow を介した action-free 事前学習**の恩恵が大きい。ego-motion と他者の動きが両方 flow に写る点も運転向き
- world model の制御インタフェースとして「未来の flow を条件付けに使う」のは、テキストや数値 action より運転シーンの微細な動き (割り込み・減速波) を指定しやすい可能性がある
- 実験アイデア: 手持ちの動画予測系 world model に flow stream を追加し、(1) action-free 走行動画での事前学習の有無、(2) 数値 action 条件付け vs flow 条件付け、で未来予測の軌道精度を比較。flow 抽出は既製の推定器 (RAFT 等) で足りるのでデータ準備コストは低い
