# MBTI: Multi-Branch Efficient Fine-Tuning for Hyperspectral Foundation Models

**arXiv:** 2607.12782v1 | **topic:** fm_distill_finetune (P2) | **score:** 4/5

## 何を解決する論文か

hyperspectral (数十〜数百の波長帯を持つ) 画像の基盤モデルは、**センサごとにバンド構成 (チャネル数・波長配置) が違う**ため下流タスクへの転移が難しい。既存手法は入力スペクトルを圧縮・選択・整形してモデルの入力仕様に合わせるが、情報を捨てて局所的な連続性を壊す。MBTI は全バンドの情報を保ったまま、少ラベルの下流分類へ効率的に適合する multi-branch な PEFT (Parameter-Efficient Fine-Tuning; 大半のパラメータを凍結し少数だけ学習する適合手法) フレームワークを提案する。

## 手法の核心

- 入力を**連続した波長帯のサブセットに分割して複数 branch に流す**前処理。端数のバンドは band reuse (既存バンドの再利用) で埋め、無意味な padding とスペクトル情報の損失を回避
- 各 branch に**独立の LoRA (Low-Rank Adaptation; 低ランク行列だけを学習する PEFT 手法)** を挿入し、波長帯ごとにタスク特化の特徴を学ばせる。事前学習パラメータは凍結
- multi-branch channel attention fusion で全 branch の特徴を再較正して統合
- rank-8 構成で学習対象は全パラメータの約 2.3% のみ。3つの公開データセットで代表手法と同等以上

## P2 への適用可能性・実験アイデア

- P2 の「**他機種への適合**」問題の縮図: 車種・世代でカメラ構成やセンサチャネルが違うとき、入力を無理に共通仕様へ潰すのではなく「**センサ群ごとに branch を切り、branch 別 LoRA で適合し、attention で融合する**」という recipe はそのまま転用できる
- band reuse の発想 (足りないチャネルは近い既存チャネルで埋めて構造を保つ) は、センサ欠損・構成差のある車両への基盤モデル展開時の入力設計に使える
- 実験アイデア: マルチカメラの perception 基盤モデルを、カメラ本数・配置が異なる別車種データへ適合する際、(1) 入力を共通 view にリサンプルして単一 LoRA、(2) カメラ群ごとの branch 別 LoRA + fusion、を同一パラメータ予算で比較。少データ (数百〜数千クリップ) 設定で差が出るかを見る
