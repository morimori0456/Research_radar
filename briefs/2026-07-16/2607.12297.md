# MobileSAM2: Lightweight Segment Anything for Spatial Intelligence

**arXiv:** 2607.12297v1 | **topic:** fm_distill_finetune (P2) | **score:** 5/5

## 何を解決する論文か

動画版 segment-anything の基盤モデル SAM2 は強力だが重く、モバイル・車載級のデバイスで動かない。本論文は SAM2 を軽量 student に蒸留する recipe として HyperKD (Hypergraphical Knowledge Distillation) を提案し、画像・動画両対応の軽量ファミリ MobileSAM2 を構築する。**動画基盤モデルの蒸留で「時間方向の知識」と「粒度の異なる知識」をどう運ぶか**に正面から答えている点が新しい。

## 手法の核心

- hypergraph (1つの辺が3ノード以上を同時に結べるグラフの一般化) で teacher の知識を関係構造として明示的にモデル化し、点ごとの feature 一致ではなく**構造ごと** student に転写する
- **Temporal HyperKD**: フレームをまたぐ時間的な対応関係 (追跡・記憶に相当する知識) を hypergraph として抽出し、student を整合させる
- **Granularity HyperKD**: SAM2 が持つ多粒度 (whole / part / subpart のようなマスク粒度) の知識を hypergraph 化して蒸留する
- モデル縮小の際に HyperKD を評価関数にして student アーキテクチャを探索し、効率と精度のバランスが良い構造を選ぶ (蒸留 loss を NAS (Neural Architecture Search; アーキテクチャ自動探索) の目的に使う)
- 複数ベンチマークに加え、embodied AI タスクへの汎化まで検証

## P2 への適用可能性・実験アイデア

- 車載向けに perception 系基盤モデル (SAM 系、動画 encoder、BEV 系 teacher) を蒸留する際、**フレーム単位の feature KD だけでは時間的一貫性が student に乗らない**という問題への具体的な処方箋。capacity gap 対策としても「関係構造の蒸留」は pointwise 一致より student の容量要求が緩い
- student アーキテクチャを蒸留 loss で探索する手順は、車載チップの制約 (レイテンシ・メモリ) を制約条件に足せば、そのまま社内の蒸留パイプラインに組み込める
- 実験アイデア: 現行の動画系 teacher→student 蒸留で、(1) 通常の feature KD、(2) フレーム間対応を relation として加えた KD、を LoRA 等は固定して比較し、時系列タスク (追跡・オンライン検出) での差分を測る。hypergraph 構築は近傍 feature の類似度グラフの簡易版から始めてよい
