# Cross4D-JEPA: Dense Cross-modal Correspondence Distillation for 4D Point Cloud Representation Learning

- **arxiv-id**: 2607.00514v1 ／ **topic**: fm_distillation ／ **project**: P2 ／ **relevance**: 5/5
- **URL**: http://arxiv.org/abs/2607.00514v1

## 何を解決する論文か
LiDAR/深度センサが時系列で捉える4D点群(3D点列)の理解はロボティクス・具現化知覚の中核だが、密なアノテーションが高価で自己教師あり事前学習が必要。既存のpretextタスクはほぼ**モダリティ内**で、2D基盤モデルから知識移転する少数手法もクリップごとに**単一のグローバル埋め込み**しか使わず、基盤モデルが計算する豊かなパッチごとの意味を捨てている。本論文は凍結2D基盤モデルを4D点エンコーダへ**密に蒸留**することでこのギャップを埋める。

## 手法の核心
- **teacher-student密蒸留**: 凍結した2D基盤モデル(画像: DINOv2 / 動画: V-JEPA 2)を教師に、4D点エンコーダを生徒として蒸留。
- **dense cross-modal correspondence**: 各3D点を、それが投影される教師パッチ特徴に対応付ける(グローバル集約せず点ごとに対応)。
- **per-point目的関数**: 生徒に潜在空間で教師特徴を一致させるよう学習。**masking/negatives/decoderが不要**でレシピが単純。
- **granularityが効く**: 性能向上の主因は教師のモダリティより「対応の粒度(密対応)」であると分析で帰結。
- **結果**: MSR-Action3D / DeformingThings4D / NTU-RGB+D 60 / HOI4D の4ベンチでintra-modal・global cross-modalベースラインを一貫して上回り、重量級手法と競合。ドメイン間転移・ラベル効率・full-label FTすべて改善。**13x小さいエンコーダが重量級pooling backboneに匹敵**。

## P2(Foundation Model 蒸留)への適用可能性
- **capacity gap対策の実例**: 大きな凍結FMを13x小さいエンコーダへ落として同等性能 → 自社の蒸留で狙う「大教師→小生徒」設定の直接の参考。
- **レシピが実務的**: negatives/decoder/momentum不要で学習が安定。蒸留パイプラインの実装・チューニングコストが低い。
- 「グローバル埋め込みでなく密なパッチ/点対応で蒸留する」という一般原則は、点群に限らず自社の他モダリティ蒸留(画像・特徴マップ)にも移植可能。

### 試すべき実験アイデア
1. 自社の点群/センサ生徒モデルに、既存のグローバル蒸留 vs 密対応蒸留を差し替えて**ラベル効率(少数ショット時の精度)**を比較。粒度がどれだけ効くかを再現。
2. 教師をDINOv2 vs V-JEPA2 vs 自社FMで振り、"教師のモダリティより粒度が支配的"という主張が自社データでも成立するか検証。
3. 生徒の縮小率を1x/4x/13x/25xとスイープし、capacity gapが破綻する境界(精度が急落する圧縮率)を特定 → 自社の蒸留サイズ選定表に反映。
