# Video-Oasis: Rethinking Evaluation of Video Understanding

- **arXiv**: 2603.29616 | **published**: 2026-07-01 | **hf_upvotes**: 51
- **topic**: sns_wildcard (探索枠) | **score**: 4/5

## 何を解決する論文か

Video-LLM (動画を入力に取る large language model) のベンチマークスコアが「本当に動画を見て解いた結果なのか、言語知識だけで解けてしまっているのか」を切り分けられない、という評価の根本問題に取り組む。新ベンチマークを足すのではなく、既存ベンチマーク群を体系的に監査 (audit) する診断スイート Video-Oasis を提案し、既存サンプルの 55% が視覚入力も時間文脈もなしで解けてしまう shortcut (モデルが本来の能力を使わずに正解へ到達できる抜け道) であることを示した。

## 手法の核心

- **ベンチマークを作らず、ベンチマークを監査する**: 「また新しいベンチマーク」ではなく、既存ベンチマークの各サンプルに対して blind evaluation (視覚入力を与えずに解かせる) と temporal ablation (フレーム順序・時間文脈を壊して解かせる) を適用し、shortcut で解けるサンプルを特定・除去する診断プロトコルを整備。
- **55% が shortcut で解ける**という定量的発見。スコアの過半が視覚・時間理解を測っていなかったことになる。
- **フィルタ後の video-native な難問だけで再評価すると、SOTA モデルはランダム推測をわずかに上回る程度**。既存リーダーボードの序列が実力を反映していない可能性を示す。
- 除去後に残った難問集合を testbed として、どの設計選択 (アーキテクチャ・学習法) が頑健な動画理解に効くかを切り分けて検証する、という「診断 → 設計指針」の流れまで含む。
- 診断スイートは sustainable (新ベンチマーク登場のたびに再適用可能) な作りで、コード公開あり: https://github.com/sejong-rcv/Video-Oasis

## プロジェクトへの適用可能性

**P1 (Planner AI + 評価) — 最も直接的**。この論文の構造は自動運転の open-loop planning 評価で知られる問題 (nuScenes の open-loop 評価が ego status — 自車の速度・過去軌跡などの状態情報 — だけでほぼ解けてしまい、知覚を見ていなくてもスコアが出る) と完全に同型。監査プロトコルをプランナー評価にそのまま輸入できる。

- **実験アイデア (P1)**: 自前のプランナー評価セットに blind evaluation を適用する。知覚入力 (カメラ/BEV特徴) を落とす・履歴だけ与える、の2条件で各シナリオを解かせ、「知覚なしで解けるシナリオ」の割合を測る。その割合が高ければ、shortcut シナリオを除去した「video-native 相当」のハード評価サブセットを切り出し、以後のプランナー比較はそちらで行う。
- **実験アイデア (P1)**: ベンチマーク刷新の議論をする際、Video-Oasis の「監査 → フィルタ → 残った難問で設計比較」の3段構成を評価手法提案のテンプレートとして使う。

**P3 (VLA / World Model / E2E)**: VLA (vision-language-action; 視覚と言語の入力から行動を出力するモデル) を QA 形式や言語出力で評価する場合、言語 prior (学習済みの言語知識だけで答えを当てる傾向) による水増しが同じ形で起こる。VLA 評価セットに blind ablation を入れて「動画を見ないと解けない設問」の比率を管理するのは低コストで効果が大きい。

**P2 (蒸留 + fine-tuning)**: 蒸留した student の評価に shortcut だらけのベンチマークを使うと、「teacher の言語 prior の模倣」と「視覚理解の保持」を区別できない。shortcut 除去済みサブセットを蒸留前後の regression テストに使えば、蒸留で視覚側の能力だけが落ちる劣化を検出できる。
