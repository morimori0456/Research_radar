# Weak-to-Strong Generalization via Direct On-Policy Distillation (Direct-OPD)

**arXiv:** 2607.05394 | **topic:** sns_wildcard (EXPLORE) | **score:** 5/5 | **hf_upvotes:** 91

> wildcard 枠(分野外だが注目度が高い探索枠)での採用。実際には蒸留手法であり P2 に直結するため、視野拡張と本線の両方に効くと判断。

## 何を解決する論文か

RLVR (Reinforcement Learning with Verifiable Rewards; 正誤が自動検証できる報酬で LLM の推論を強化する学習)は強力だが、新しい強モデルが出るたびに繰り返すのが高価。ターゲットモデル自身が学習中に大量の rollout (試行生成)を出す必要があるからで、モデルが大きくなるほど post-training がボトルネックになる。本論文は **小さいモデルで RL を回し(rollout が安い)、そこで学んだものを再利用して強いモデルを改善する** weak-to-strong の方法を提案する。

## 手法の核心

- post-RL の weak teacher をそのまま蒸留するのは不十分。teacher の最終ポリシーは「有用な RL の学び」と「小モデルゆえの限界」を混ぜてしまうため
- **Direct-OPD は teacher の RL による "ポリシーの変化分" だけを転送する**: post-RL teacher と、その RL 前の reference (同じ小モデルの学習前チェックポイント)を比較し、両者の log 比を student への dense な implicit reward (密な暗黙報酬)として扱う
- 平たく言えば「チェックポイントのペアが、RL によって weak モデルがどの行動をより/より少なく取るようになったかを教える」→ その信号を強い student 自身の on-policy な状態に適用する
- これで、ターゲットモデル上で疎報酬 RL を回さずに weak モデルの RL 教師信号を再利用できる
- 実証: Qwen3-1.7B を AIME 2024 で 48.3% → 58.3% に、8×A100 でわずか 4時間。step を揃えた直接 RL を上回り、複数のポリシー変化を逐次合成できる

## 追っているプロジェクトへの適用可能性・実験アイデア

- **P2 (蒸留/適合) に直結**: これは「新しい loss / capacity gap 対策」の一種で、しかも通常の logits/feature 蒸留と異なり **teacher の学習前後ペアから "変化分" を蒸留する** という新しい定式化。P2 の recipe 引き出しに加えたい
- 一般的示唆として、蒸留を「最終状態のコピー」ではなく「学習で獲得した差分の転送」と捉える視点は、driving ポリシー/プランナーの世代交代(小モデルで安全に試した改善を大モデルへ移す)にも応用しうる
- 実験アイデア: 小さいポリシー/推論モデルで検証可能報酬 RL を回し、(pre-RL, post-RL) ペアの log 比を dense reward として大きい student に適用。(a) post-RL teacher の直接蒸留、(b) Direct-OPD、(c) student 直接 RL を計算予算を揃えて比較
- リスク確認: reference チェックポイントの選び方(どの時点を "RL前" とするか)に結果が敏感か、複数ポリシー変化の逐次合成が干渉なく積めるかを ablation
