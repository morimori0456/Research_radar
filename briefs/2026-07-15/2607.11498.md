# See like a Robot: Robot-Centric Pointmaps for Vision-Language-Action Models

**arXiv:** 2607.11498v1 | **topic:** next_arch (P3) | **score:** 4/5

## 何を解決する論文か

VLA (Vision-Language-Action; 視覚観測と言語指示からロボット action を直接予測するモデル) が出力する action はロボット自身の 3D 座標系で定義されるのに、多くの VLA はシーンをカメラ座標系で観測する。この **frame mismatch (観測と行動で座標系がずれている問題)** は、視点が固定なら policy が単一の観測→action 対応を丸暗記できるため無害だが、多様なカメラ設置のデータを大規模に混ぜると、この対応を視点をまたいで汎化する必要があり難しくなる。本論文は robot-centric pointmap でこのずれを解消する。

## 手法の核心

- **robot-centric pointmap**: 各ピクセルにそのシーン点の 3D 座標を「ロボット座標系で」格納した画像。カメラ座標ではなくロボット座標で幾何を持つ
- 事前学習済み 2D VLA が期待する dense な H×W グリッド形状を保つため、既存 VLA に最小限のアーキ変更で統合できる(pointmap を追加入力チャネルとして与えるイメージ)
- RoboCasa で pi0.5 と SmolVLA の両方を改善し、カメラ視点ベース・3D-aware の代表的ベースラインを上回る
- 実機実験: **学習時に見ていないカメラ配置**にカメラを動かすと、RGB のみのポリシーに対する優位が拡大 = 視点汎化に効いている

## P3 への適用可能性・実験アイデア

- P3 の VLA 構造への直接の設計示唆。自動運転は本質的に **マルチカメラ + ego-vehicle 座標系** で、観測(各カメラ)と行動(ego 座標の軌道/制御)の frame mismatch がまさに存在する。ロボット座標を ego 座標に読み替えれば同じ処方箋が使える
- 「dense グリッド形状を保つので事前学習 2D バックボーンに最小改修で足せる」という性質は、既存の driving VLA / E2E モデルへの後付け改善として魅力的
- 実験アイデア: 手元の E2E/VLA 運転モデルに、各カメラの depth (or LiDAR) から作った **ego-frame pointmap** を追加入力し、(a) RGB のみ、(b) +camera-frame pointmap、(c) +ego-frame pointmap で軌道予測/制御を比較。特に **学習時と異なるカメラ外部パラメータ(取り付け位置変更)** でのゼロショット汎化を測ると、車種・センサ構成をまたぐ転用(P2 的な適合)にも効くか分かる
- pointmap の供給元(単眼 depth 推定 vs ステレオ vs LiDAR 投影)の質が下流性能にどう効くかの ablation
