# SAKD: Shift-Augmented Knowledge Distillation

**arXiv:** 2607.11557v1 | **topic:** fm_distill_finetune (P2) | **score:** 4/5

## 何を解決する論文か

KD (Knowledge Distillation; 大きな teacher モデルの出力を小さな student に模倣させて圧縮する手法) は単一 teacher の固定的な視点に依存し、教師信号の多様性が乏しい。multi-teacher 蒸留は多様性を足せるが計算・保存コストが高い。既存の「単一 teacher から仮想的な複数ビューを作る」手法は、ランダム摂動 (効率的だが多様性が制御できない) か構造化 augmentation (多段学習でパラメータが線形増加) のトレードオフに縛られる。SAKD は teacher の静的特徴ではなく **student の学習途中の特徴** を摂動生成の条件に使うことで、このトレードオフを解消する。

## 手法の核心

- 既存手法の弱点を「teacher の強いが静的な特徴でビューを生成している」ことに帰着させ、代わりに **進化し続ける student 特徴** を動的な条件として使う
- パラメータフリーの cyclic shift (特徴を巡回シフトするだけの操作) で摂動を生成 → 追加パラメータなし・単一ステージ学習
- 生成されるビューが student の状態に適応するため、多様かつ制御された教師信号になる
- CIFAR-100 / ImageNet で、ランダム摂動法を一貫して上回り、2段階手法と同等精度を、大幅に少ないパラメータかつ事前学習なしで達成

## P2 への適用可能性・実験アイデア

- P2 の「再現可能な蒸留 recipe / capacity gap 対策」に直球で該当。**追加パラメータ・多段学習なし**という制約は、蒸留を実務で回す上で効いてくる(実装が既存 KD ループへの小改修で済む)
- capacity gap (teacher と student の容量差が大きいと蒸留が効きにくい問題) に対し、student 適応的なビューが効くという主張は、大きな基盤 teacher → 小さなエッジ student という P2 の典型構図で検証したい
- 実験アイデア: 手元の KD パイプライン(logits/feature 蒸留)に cyclic-shift ビュー生成を挿入し、(a) ベース KD、(b) ランダム摂動、(c) SAKD を同一 student で比較。特に teacher/student の容量比を変えたときの gap 耐性を測る
- 応用: 分類だけでなく回帰・検出 head でも cyclic shift が有効か(特徴のチャネル巡回が空間タスクで意味を保つか)を確認する価値あり
