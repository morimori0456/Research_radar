# Cross4D-JEPA: Dense Cross-modal Correspondence Distillation for 4D Point Cloud Representation Learning

- **arxiv-id**: 2607.00514v1 ／ **topic**: fm_distillation ／ **project**: P2 ／ **relevance**: 5/5
- **URL**: http://arxiv.org/abs/2607.00514v1

## 何を解決する論文か
動的4D点群(LiDAR/depthの時系列3D点)の理解はロボティクス・embodied知覚の中核だが、密なアノテーションが高コストで自己教師あり事前学習が現実解になる。既存のpretextはほぼintra-modalで、2D基盤モデルから知識移転する数少ない手法もclip1つあたり単一のグローバル埋め込みしか使わず、per-patchの豊かな意味を捨てている。本論文は**凍結2D基盤モデルを4D点エンコーダへ密に蒸留**する teacher-student 法を提案する。

## 手法の核心
- **密なcross-modal対応**: 全ての3D点を、それが投影される teacher パッチ特徴に対応づける(グローバル埋め込みでなくパッチ粒度)。
- **per-point目的関数**: 生徒が latent 空間でこれらの特徴に一致するよう学習。**masking / negatives / decoder 不要**でレシピが単純。
- **teacher自由度**: 凍結画像モデル DINOv2 / 動画モデル V-JEPA 2 のいずれでも蒸留可能。利得の主因は teacher のモダリティよりも**対応の粒度**であると分析。
- **capacity gap実証**: **13x小さいエンコーダが重量級プーリングbackboneに匹敵**。ドメイン跨ぎ転移・ラベル効率・full-label fine-tuning のいずれも改善。
- **検証**: MSR-Action3D / DeformingThings4D / NTU-RGB+D 60 / HOI4D の4ベンチでintra-modal・global cross-modalベースラインを一貫して上回る。

## P2(Foundation Model 蒸留)への適用可能性
- **本命**。「グローバル埋め込みでなく密な対応で蒸留」+「13x圧縮で同等」は、自社の大教師→小生徒蒸留にそのまま効く汎用原則。capacity gap 対策の再現可能な recipe。
- decoder/negatives/momentum 不要のため既存パイプラインへ最短で移植でき、実装リスクが低い。
- LiDAR時系列を扱うP1(AV知覚)とも接点があり、蒸留した軽量4Dエンコーダを車載推論に載せる筋がある。

### 試すべき実験アイデア
1. 自社の大教師モデルを凍結し、既存の**グローバル蒸留 vs 密対応蒸留**を差し替えてラベル効率(少ラベルfine-tune精度)を比較。
2. 「利得は粒度が主因」の主張を自社データで検証: パッチ粒度を粗↔細で振り、精度と学習コストの関係を測る。
3. teacher を画像(DINOv2)/動画(V-JEPA2)で入れ替え、自社ドメインでどちらが効くかと、13x圧縮時の精度劣化カーブを描く。
