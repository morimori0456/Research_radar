## 4. 記憶チェック (5問)

1. **Q**: WM Admissibility が2つの driving world model で実証した「逆転」とは?
   **A**: **FVD 等の視覚生成品質で上位のモデルが、行動追従性(入力した行動どおりに世界が応答するか)では下位**。見た目の指標を world model の信頼性の代理にしてはいけない。
2. **Q**: ZipDepth (6.1M、teacher の 1/50) が zero-shot 汎化をほぼ保てた鍵は?
   **A**: 蒸留手法や teacher の質ではなく **multi-domain の大規模データで教える**こと。蒸留の頑健性は「何を教えるか」より「蒸留データの分布の広さ」で決まる。
3. **Q**: Soft Clamp が扱う「loss は健全なのに挙動が歪む」現象の原因と処方は?
   **A**: モード切替を担う**少数 token の leverage 不均衡**(その token が生成全体を支配)。per-token の発散を soft に圧縮する校正で、精度を保ったまま挙動シフトを修正。
4. **Q**: PixelPilot が同時に解いた VLA の2つの急所と、その一手は?
   **A**: カメラパラメータ結合で**異種データ混合学習がスケールしない**問題と、ego-status だけで軌跡を出す**近道学習**。軌道予測を**画像平面の 2D タスクに再定式化**し 3D 化は推論時に回す。
5. **Q**: MIPI/MIPU が指摘する LLM RL 崩壊の主因と、蒸留・PEFT にも効く一般原理は?
   **A**: **training-inference mismatch**(学習エンジンと推論エンジンが同一系列に別の確率を返す)。一般原理は「**最適化している対象と価値が出る対象 (deploy 時挙動) のズレを疑え**」。
