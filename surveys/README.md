# Living Surveys — テーマ別に育つ個人サーベイ

> Karpathy の personal wiki 方式: 「LLM がほぼ全てを書き、本人はほぼ全てを読む」。
> 日次ブリーフ (briefs/) は日付順に流れて死蔵されるが、ここはテーマ別に**蓄積**される。
> 週次レビューが毎週、今週のブリーフを該当サーベイに統合する。

## 各サーベイの構造 (週次レビューはこれに従って更新する)

```markdown
# <テーマ名> — Living Survey
最終更新: YYYY-MM-DD

## 一言でいうと
この分野は何を解こうとしているか (3行以内。更新のたびに磨く)

## 系譜マップ
主要なアプローチの分岐と代表手法 (箇条書きの木構造)

## 重要論文リスト
| 日付 | 論文 | 一言 | brief |
(新しいものを上に追加。briefへの相対リンク付き)

## Open Questions
未解決の問い / 自分が論文にできそうな隙間

## 自分の実験・メモ
(本人が自由に書く欄。週次は消さない)
```

## 運用ルール

- テーマの新設は週次レビューが判断する (同じテーマのブリーフが3本溜まったら新設)
- 「一言でいうと」と「系譜マップ」は毎回**上書きで磨く** (追記で伸ばさない)
- 「自分の実験・メモ」欄だけは本人の領域。ループは書き換えない
- 数ヶ月育ったサーベイは、そのまま ML_report のレポートや論文の related work の下書きになる
